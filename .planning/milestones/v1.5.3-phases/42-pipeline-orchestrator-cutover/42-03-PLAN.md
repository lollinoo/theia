<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Scheduler task -> collector worker | Untrusted device/network I/O enters the runtime through scheduled work items |
| State store -> WebSocket payload | Internal runtime state becomes client-visible snapshot and delta messages |
| Topology notify -> broadcast loop | Topology changes must not be broadcast before the corresponding snapshot is available |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-09 | Denial of Service | scheduler completion path | mitigate | Every worker completion reports back to the scheduler so in-flight slots cannot leak and stall future polls. Severity: HIGH if unmitigated because the pipeline deadlocks after enough tasks. |
| T-42-10 | Tampering | snapshot builder contract | mitigate | Snapshot/delta helpers preserve the exact current section names and status-string domain, preventing incompatible payload drift during cutover. Severity: HIGH. |
| T-42-11 | Repudiation | topology notify ordering | mitigate | Broadcast loop drains `topologyNotify` only after snapshot/delta send, with explicit tests to lock the ordering. Severity: HIGH. |
| T-42-12 | Information Disclosure | snapshot getter aliasing | mitigate | `GetSnapshot()` returns a clone of the last snapshot so callers cannot mutate broadcast state. Severity: MEDIUM. |

Security gate status: ASVS L1, block_on=high. No HIGH-severity issue remains once runtime, completion, and ordering tests pass.
</threat_model>

    - `DeviceState` grows a `LinkMetrics []domain.LinkMetrics` field for live overview throughput state
    - `StateUpdate` grows `VolatilityClass` and `LinkMetrics`
    - `Store.Update()` uses `VolatilityClass` to decide what can mutate:
      - `performance`: merge CPU/memory/temperature and `LinkMetrics`; preserve existing uptime if the performance sample omits it
      - `operational`: update reachability, consecutive failures, timestamps, and uptime only; never clear CPU/memory/temperature or `LinkMetrics`
      - `static`: do not overwrite live overview metrics or link metrics; only refresh scheduler metadata needed by staleness bookkeeping
    - failed performance polls preserve the last-known metrics and link metrics; they do not zero those fields
    - `Snapshot()` and `GetDevice()` deep-copy `LinkMetrics`
    - diff detection treats meaningful link-metric changes the same way it treats device-metric changes
  </behavior>
  <action>

    1. Extend `DeviceState` with:

       ```go
       VolatilityClass domain.VolatilityClass
       LinkMetrics     []domain.LinkMetrics
       ```

    3. Refactor `Store.Update()` into explicit tier-aware helpers:
       ```go
       func applyPerformanceUpdate(next *DeviceState, update StateUpdate)
       func applyOperationalUpdate(next *DeviceState, prev DeviceState, update StateUpdate)
       func applyStaticUpdate(next *DeviceState, update StateUpdate)
       ```

    4. Implement the exact merge rules:
       - `performance` success:
         - copy `CPUPercent`, `MemPercent`, `TempCelsius`, and `CollectedAt` from `update.Metrics` when non-nil
         - keep `UptimeSecs` from the existing state unless `update.Metrics.UptimeSecs` is non-nil
         - replace `LinkMetrics` only when `len(update.LinkMetrics) > 0`
         - recompute health from the merged metric set
       - `performance` failure:
         - preserve the existing metric and link values
         - still record timestamp / interval metadata for staleness bookkeeping
         - do not reset severities or set metrics to empty
       - `operational` success/failure:
         - use the existing reachability state machine for `PollSuccess`
         - update only `UptimeSecs` from `update.Metrics`
         - leave `CPUPercent`, `MemPercent`, `TempCelsius`, and `LinkMetrics` untouched
       - `static`:
         - never clear or replace live overview metrics or link metrics

    5. Add store helpers:
       ```go
       func cloneLinkMetrics(in []domain.LinkMetrics) []domain.LinkMetrics
       func linkMetricsEqual(a, b []domain.LinkMetrics) bool
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Collector result -> state store | Raw per-tier polling output crosses into the in-memory source of truth used for WebSocket delivery |
| Store snapshot -> broadcaster | Read-only snapshots must not alias mutable runtime slices or maps |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-42-01 | Tampering | tier merge logic | mitigate | Tier-aware helpers prevent operational polls from overwriting CPU/memory/temperature or link metrics with partial data. Severity: HIGH if unmitigated because overview metrics disappear during normal polling. |
| T-42-02 | Denial of Service | failed performance handling | mitigate | Preserve last-known metrics on failed performance polls so transient SNMP misses do not collapse the live overview into empty data. Severity: HIGH if unmitigated because repeated transient failures create user-visible flapping. |
| T-42-03 | Information Disclosure | snapshot cloning | mitigate | Deep-copy `LinkMetrics` on `Snapshot()` and `GetDevice()` so callers cannot mutate in-memory runtime state through shared slices. Severity: MEDIUM. |
| T-42-04 | Repudiation | cross-tier regression tests | mitigate | Focused tests assert the exact cross-tier behaviors that previously regressed implicitly. Severity: MEDIUM. |

Security gate status: ASVS L1, block_on=high. No HIGH-severity threat remains open once the tier-aware merge tests pass.
</threat_model>

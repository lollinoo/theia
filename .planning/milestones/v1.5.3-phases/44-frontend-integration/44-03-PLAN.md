## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| websocket snapshot -> DeviceCard UI | Backend-owned live state is translated into user-visible status signals on the canvas |
| local stale timer -> card metadata | Frontend fallback logic can either preserve or accidentally erase backend status metadata |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-44-09 | Tampering | `frontend/src/components/DeviceCard.tsx` | mitigate | Primary dot/glow and health label must map from `metrics.health`, not from raw metrics, reachability, or alert-driven composites. Severity: HIGH. |
| T-44-10 | Denial of Service | `frontend/src/components/canvas/useCanvasData.ts` stale timer | mitigate | Blank only numeric metrics on local stale fallback so Phase 44 metadata stays visible during reconnect periods. Severity: MEDIUM. |
| T-44-11 | Information Disclosure | card metadata row | accept | Freshness and cadence reveal only polling state already intended for operators; no new sensitive data is exposed. Severity: LOW. |
| T-44-12 | Repudiation | freshness tier rendering | mitigate | Pure helper tests lock the `Fresh` / `Stale` / `Dead` thresholds and waiting-for-first-poll copy so the UI cannot drift silently. Severity: LOW. |

Security gate status: ASVS L1, block_on=high. The plan blocks on Task 2 because using non-health signals for the primary card state would leave a HIGH-severity integrity/tampering risk unresolved.
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Canvas panel lifecycle -> websocket control messages | UI state changes become socket commands here |
| Browser reconnect -> server subscription state | Reconnect can duplicate or lose subscriptions if not handled carefully |
| Incoming snapshot payload -> frontend state atom | Optional detail fields from the backend become client state here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-11 | Tampering | `DeviceMetricsDTO` parser | mitigate | Parse optional detail fields explicitly in `metrics.ts` so malformed data does not poison the shared snapshot atom. Severity: MEDIUM. |
| T-43-12 | Denial of Service | reconnect re-subscribe path | mitigate | Keep one socket, one `detailDeviceIdRef`, and one `lastSubscribedDeviceIdRef` so reconnect sends at most one subscribe for the current device. Severity: HIGH. |
| T-43-13 | Information Disclosure | panel ownership logic | mitigate | `getCanvasDetailDeviceId()` returns a device ID only for device-scoped canvas panels; link panels and other surfaces always resolve to `null` per D-03 to D-05. Severity: HIGH. |
| T-43-14 | Repudiation | close/switch cleanup | mitigate | Canvas derives subscription state directly from `panelContent`, so every close/switch path that already nulls or replaces `panelContent` triggers an immediate unsubscribe. Severity: HIGH. |
| T-43-15 | Elevation of Privilege | duplicate frontend state paths | mitigate | Keep `useWebSocket()` as the only socket and snapshot owner; do not add panel-local detail caches or alternate sockets. Severity: HIGH. |

Security gate status: ASVS L1, block_on=high. No HIGH-severity issue remains once lifecycle helper and reconnect tests pass.
</threat_model>

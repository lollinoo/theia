## Cross-Phase Integration (31/34 wired clean)

### Integration Issues

**INT-01: AreaHub "Open Settings" CTA does not open Settings panel** (medium)
- `AreaHub.tsx` calls `onOpenSettings()` which is wired to `setActiveView('canvas')` in `App.tsx`
- User lands on Canvas view but Settings panel does not open -- must manually click gear icon
- Affects: AREA-05, AREA-07
- File: `frontend/src/App.tsx:78-80`

**INT-02: Stale area state after Settings area mutation** (low-medium)
- `App.tsx` only re-fetches areas on mount and when `activeView === 'hub'`
- After creating/editing area in Settings, NavigationPill area buttons and Dashboard area filter don't update until Hub view is visited
- Affects: AREA-05, AREA-09
- File: `frontend/src/App.tsx:34-40`

**INT-03: DeviceConfigPanel raw neon colors in light theme** (cosmetic)
- Lines 330, 338 use `areas.find(...)?.color` directly without `adaptAreaColor`
- Other components (DeviceRow, AreaCard, Canvas) correctly use `adaptAreaColor`
- Affects: THEME-05
- File: `frontend/src/components/DeviceConfigPanel.tsx:330,338`

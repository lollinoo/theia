**frontend/src/hooks/useAutoLayout.ts:** Force-directed layout function (not a hook with side effects -- a pure function that computes positions):

```typescript
export function computeForceLayout(
  nodes: {id: string; x?: number; y?: number; pinned?: boolean}[],
  edges: {source: string; target: string}[],
  width: number,
  height: number
): Map<string, {x: number; y: number}>
```

Use d3-force simulation with:
- forceLink: connects linked nodes (distance ~200)
- forceManyBody: repulsion (strength ~ -300) to spread nodes
- forceCenter: center of provided width/height
- forceCollide: prevent overlap (radius ~80)
- Run simulation synchronously (tick 300 times) rather than animated -- we want instant results
- IMPORTANT: Pinned nodes should have their positions fixed (set fx, fy in d3-force). Unpinned nodes get computed positions.

**frontend/src/hooks/usePositions.ts:** Hook for position persistence:

```typescript
export function usePositions() {
  // fetchPositions(): Promise<Map<string, {x, y, pinned}>> — GET /api/v1/positions
  // savePositions(positions: {device_id, x, y, pinned}[]): Promise<void> — PUT /api/v1/positions
  // Returns { positions, loading, fetchPositions, savePositions }
}
```

Key behaviors:
- On load: fetch saved positions from API
- On save: debounce 1 second, then PUT all current positions to API
- When a node is dragged: mark it as pinned=true, trigger debounced save

**frontend/src/components/Canvas.tsx:** Update the existing Canvas to integrate layout and persistence:

1. On mount: fetch devices, links, AND positions (all 3 in parallel)
2. Merge positions with devices:
   - If a device has a saved position -> use it (and set pinned from saved data)
   - If a device has NO saved position -> leave position undefined (will be auto-laid-out)
3. Run computeForceLayout on all nodes, passing saved positions as pinned fixed points
4. Apply computed positions to nodes
5. On node drag end (onNodeDragStop):
   - Mark the dragged node as pinned=true in local state
   - Trigger debounced savePositions with ALL current node positions
6. Save all positions after initial auto-layout (so new devices get positions persisted)
7. Replace the random positioning from Plan 03 with this layout system

The flow: Load data -> Merge saved positions -> Force layout (respecting pins) -> Render -> On drag, save

IMPORTANT: Do NOT re-run layout on every render. Only run layout once on initial data load or when new devices appear. Use a ref to track whether initial layout has been applied.

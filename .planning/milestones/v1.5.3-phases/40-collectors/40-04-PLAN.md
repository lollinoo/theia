Create `internal/collector/prometheus.go` with:

```go
type PrometheusEnrichmentClient interface {
    QueryHostnames(ctx context.Context, labelName string, labelValues []string) (map[string]string, error)
    QueryProbeStatus(ctx context.Context, deviceIPs []string) (map[string]bool, error)
    QueryAlerts(ctx context.Context) ([]domain.AlertState, error)
}

type PrometheusCollector struct {
    client PrometheusEnrichmentClient
    now    func() time.Time
}

func NewPrometheusCollector(client PrometheusEnrichmentClient) *PrometheusCollector
func ResolvePrometheusLabel(device domain.Device) (labelName string, labelValue string, ok bool)
func (c *PrometheusCollector) CollectDeviceEnrichment(ctx context.Context, device domain.Device) (PrometheusEnrichment, error)
func (c *PrometheusCollector) CollectAlerts(ctx context.Context, devices []domain.Device) (map[uuid.UUID][]domain.AlertState, error)
func MapAlertsToDevices(devices []domain.Device, alerts []domain.AlertState) map[uuid.UUID][]domain.AlertState
```

Implementation rules:
- `CollectDeviceEnrichment`:
  - calls `ResolvePrometheusLabel`
  - uses `QueryHostnames` with a single label value when available
  - uses `QueryProbeStatus` with `device.IP` when available
  - returns `PrometheusEnrichment{DeviceID: device.ID, Hostname: ..., ProbeReachable: ..., CollectedAt: c.now().UTC()}`
  - does NOT call `QueryAlerts`; alert collection is batch-oriented via `CollectAlerts`
- `CollectAlerts`:
  - calls `QueryAlerts`
  - delegates grouping/mapping to `MapAlertsToDevices`
- `MapAlertsToDevices` should preserve deterministic ordering: device UUID, then severity, then alert name.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-13 | Tampering | Prometheus enrichment data | mitigate | Narrow the collector-facing interface to hostname/probe/alert methods only so enrichment cannot override SNMP core metrics by accident. |
| T-40-14 | Information Disclosure | alert mapping helper | accept | Alert content is already returned by Prometheus; this helper only groups it by in-process device UUID for later consumers. |
| T-40-15 | Spoofing | label resolution | mitigate | `ResolvePrometheusLabel` uses existing device metadata with explicit fallback rules and returns `ok=false` when no safe query target exists, avoiding accidental wildcard queries. |
| T-40-16 | Denial of Service | future scope creep back to Prometheus-primary | mitigate | Acceptance criteria and interface shape explicitly forbid `QueryDeviceMetrics`, `QueryLinkMetrics`, and `QueryInterfaces` in this phase. |
</threat_model>

<objective>
Implement hash-based change detection in MetricsCollector and add the `snapshot_delta` message type to the WebSocket layer. After each collection cycle, each section of the snapshot is hashed per device_id. Only entries whose hash changed since the previous cycle are included in a sparse `SnapshotPayload` broadcast as a `snapshot_delta` message. If nothing changed, the broadcast is skipped entirely.

Purpose: Reduce WebSocket payload from ~55KB per cycle (77+ devices) to only the changed entries, dramatically cutting bandwidth when most devices report stable metrics.

Output: Modified `messages.go` with new constant, modified `metrics_collector.go` with `prevHashes` map and delta logic, new test coverage for hash computation and delta behavior.
</objective>

    currentHashes := computeSnapshotHashes(snapshot)

    // Build delta against previous hashes
    c.mu.Lock()
    prev := c.prevHashes
    c.prevHashes = currentHashes
    c.mu.Unlock()

    if prev == nil {
        // First cycle: send full snapshot (same as current behavior)
        c.hub.Broadcast(ws.Message{
            Type:    ws.MessageTypeSnapshot,
            Payload: snapshot,
        })

            // Per D-03: send sparse payload as snapshot_delta

        // Per D-05: if delta is nil, skip broadcast entirely

  <action>
Create four new files:

**ContextMenu.tsx:** A generic dark-themed context menu component.
- Props: `items: Array<{label: string, onClick: () => void, variant?: 'danger'}>`, `position: {x: number, y: number}`, `onClose: () => void`
- Renders a fixed-position menu at the given coordinates with dark styling matching the existing edge context menu in Canvas.tsx (rounded-xl, border-border-subtle, bg-bg-surface/95, backdrop-blur-xl, shadow)
- Each item is a button with hover highlight (bg-bg-elevated), danger variant uses text-status-down
- Closes on click outside or Escape (use useEffect with window event listeners)
- Menu repositions to stay within viewport bounds (if near right/bottom edge, flip direction)

**SidePanel.tsx:** A reusable slide-out panel from the right side.
- Props: `open: boolean`, `onClose: () => void`, `title: string`, `children: React.ReactNode`
- Fixed position, right side, full height, ~320px wide
- Header with title text and X close button
- Smooth slide-in/out transition (translate-x transform, 200ms ease)
- Dark theme matching app: bg-bg-surface, border-l border-border-subtle
- z-index above canvas but below context menus (z-20 for panel, z-30 for context menus)
- Escape key closes panel (but only if no context menu is open -- let parent manage this)

**ShortcutHelp.tsx:** Overlay listing all keyboard shortcuts.
- Props: `open: boolean`, `onClose: () => void`
- Centered modal overlay with backdrop blur
- Lists all shortcuts in a clean grid: shortcut key on left, action on right
- Shortcuts to list: Ctrl/Cmd+K (Search), Ctrl/Cmd+N (Add device), +/-/0 (Zoom), Ctrl/Cmd+, (Settings), ? (Shortcuts), Esc (Close)
- Display platform-aware modifier (Cmd on Mac, Ctrl otherwise) -- detect via `navigator.platform`
- Closes on Escape or clicking backdrop

**useKeyboardShortcuts.ts:** Central keyboard shortcut hook.
- Takes a map of shortcut definitions: `Record<string, {key: string, ctrl?: boolean, handler: () => void, description: string}>`
- Registers a single keydown listener on window
- Ignores shortcuts when focus is in an input/textarea/select element
- Handles Ctrl/Cmd modifier detection (metaKey on Mac, ctrlKey otherwise)
- Handles the special cases: `?` key (no modifier), `+`/`-`/`0` keys (no modifier), `Escape` key
- Returns void (side-effect only hook)

# Phase 2 -- UI Design Contract

> Visual and interaction contract for the Component Restyling phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase restyles all 25+ existing frontend components to match the Neon Topography design language. No new features or views are added -- only visual transformation using the token system from Phase 1.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom Neon Topography design system) |
| Preset | not applicable |
| Component library | none (hand-built React components with Tailwind) |
| Icon library | Material Symbols Rounded (self-hosted variable font, woff2 subset, ~15-20 icons) |
| Display font | Outfit Variable (self-hosted via `@fontsource-variable/outfit`) |
| Mono font | JetBrains Mono Variable (self-hosted via `@fontsource-variable/jetbrains-mono`) |

Source: CONTEXT.md D-01 through D-04, Phase 1 `index.css` `@theme inline` block.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, metric grid gutters |
| sm | 8px | Compact element spacing, toolbar button padding, context menu item vertical padding, context menu container vertical padding |
| md | 16px | Default element spacing, card internal padding (`px-4`), panel content padding |
| lg | 24px | Section padding, major section breaks within panels (per DESIGN.md "Spacing Level 3") |
| xl | 32px | Layout gaps, side panel width padding zones |
| 2xl | 48px | Major section breaks, vertical space around panel groups |
| 3xl | 64px | Page-level spacing (not used in Phase 2 components) |

Exceptions:
- Context menu item height: 36px (dark) / 40px (light) -- not a spacing token, a component-specific touch target.
- Toolbar button touch target: 40px x 40px (existing, retained).
- DeviceCard width: 260px (existing, retained).
- SidePanel width: 320px (existing, retained).

Source: DESIGN.md Section 6 "24px between major sections", existing component measurements, HTML mocks.

## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|

| Label | Outfit | 12px (0.75rem) | 400 (regular) | 1.4 | Section labels, uppercase tracking labels, badge text, secondary detail text (device description, vendor label subtitle), uppercase metric column headers (CPU, MEM, TEMP, UP) with `tracking-[0.16em]` |
| Title | Outfit | 20px (1.25rem) | 600 (semibold) | 1.2 | Card titles, panel headings (per DESIGN.md "Title M") |

## Glow System

Per CONTEXT.md D-08 (box-shadow only, no `backdrop-filter` on canvas nodes) and D-09 (severity-scaled intensity):

### DeviceCard Status Glow (Replaces 3px Top Border per D-11)

The DeviceCard header contains a **Glow Node**: a `rounded-full` element (10px diameter) with status-colored `box-shadow`. The old `border-t-[3px] border-tertiary` is removed entirely.

| Status | Glow Node Shadow | Card Ring |
|--------|-----------------|-----------|
| Up | `0 0 8px rgba(0,230,118, var(--nt-glow-shadow-opacity))` | `ring-1 ring-outline` (default) |
| Down | `0 0 16px rgba(255,23,68, var(--nt-glow-shadow-opacity))` | `ring-2 ring-status-down` + card-level `shadow-[0_0_28px_rgba(255,23,68,0.45)]` + `animate-pulse` |
| Probing | `0 0 12px rgba(255,234,0, var(--nt-glow-shadow-opacity))` | `ring-1 ring-outline` + `animate-pulse` |
| Unknown | `0 0 6px rgba(158,158,158, var(--nt-glow-shadow-opacity))` | `ring-1 ring-outline` |
| Degraded | `0 0 14px rgba(255,193,7, var(--nt-glow-shadow-opacity))` | `ring-2 ring-yellow-500` + card-level warning shadow |

### DeviceCard Hover Accent

Decision (Claude's discretion from CONTEXT.md D-15): Use **primary green glow** on hover for all cards regardless of status. Rationale: status glow already communicates device health; hover glow signals interactivity uniformly.

- Hover: `ring-2 ring-primary/60 shadow-[0_0_20px_rgba(0,230,118,0.15)]`
- Selected: `ring-2 ring-primary shadow-[0_0_22px_rgba(0,230,118,0.18)]`
- Highlighted (search result): `ring-2 ring-primary shadow-[0_0_28px_rgba(0,230,118,0.35)]`

### Off-Canvas Bloom (Panel Status Indicators)

Decision (Claude's discretion from CONTEXT.md D-10): Off-canvas elements (AlertsPanel status dots, future area cards) **stay box-shadow-only**. Rationale: consistency with canvas glow system, simpler CSS, and these elements are small enough that box-shadow provides sufficient visual weight without radial-blur.

Source: CONTEXT.md D-08, D-09, D-10, D-11, D-15, DESIGN.md Section 5 "Glow Node".

## No-Line Rule Enforcement (COMP-12)

The following border patterns MUST be removed and replaced with surface color tier shifts or spacing:

| Current Pattern | Replacement |
|----------------|-------------|
| `border-t border-outline` (section separators in SettingsPanel, DeviceCard) | `space-y-6` gap + surface tier shift (e.g., section on `bg-surface-high` inside panel on `bg-surface`) |
| `border-b border-outline` (SidePanel header, Toolbar button separators) | No border; use padding + surface tier contrast |
| `border-l border-outline` (SidePanel edge) | Remove; panel shadow (`shadow-panel`) provides depth |
| `border border-outline` (alert cards, search results) | Remove outer border; use `bg-elevated` + `shadow-panel` |
| `ring-1 ring-outline` (DeviceCard default) | Retained -- this is a ghost border fallback per DESIGN.md "Ghost Border Fallback" |

**Ghost border exceptions:** `ring-1 ring-outline` on DeviceCard default state and `border border-outline-subtle` on form inputs are permitted as "ghost border fallbacks" per DESIGN.md Section 4. These are functional definition borders, not layout separators.

Source: CONTEXT.md D-12, REQUIREMENTS.md COMP-12, DESIGN.md Section 2 "No-Line Rule".

## Interaction Contracts

### Theme Switching

All 26 components must respond to `data-theme` attribute changes without page reload. Transition timing:

| Property | Duration | Easing |
|----------|----------|--------|
| `background-color` | 200ms | `ease-in-out` |
| `color` | 200ms | `ease-in-out` |
| `border-color` | 200ms | `ease-in-out` |
| `box-shadow` | 200ms | `ease-in-out` |
| `backdrop-filter` | 0ms | instant (no animation on blur removal/addition) |

Implementation: Add `transition-colors duration-200` to component root elements. Shadow transitions use `transition-[box-shadow] duration-200`.

### Context Menu Separator Support

The `ContextMenu` component must support grouping items with a visual separator. Items before the separator are standard actions; items after are destructive actions. The separator is a `1px` horizontal line in `bg-outline` (dark) or `bg-glass-border` (light).

### DeviceCard Glow Node Animation

| State | Animation |
|-------|-----------|
| Up | Static glow (no animation) |
| Down | `animate-pulse` on glow node + card ring |
| Probing | `animate-pulse` on glow node |
| Unknown | Static dim glow |

Source: CONTEXT.md D-08, D-09, DESIGN.md Section 5.

## Accessibility

| Concern | Contract |
|---------|----------|
| Color contrast | All text on `bg` surfaces must meet WCAG AA (4.5:1 for body text, 3:1 for large text). Dark: `#F5F5F7` on `#161618` = 17.4:1. Light: `#1A1A1C` on `#F5F5F7` = 16.8:1. Both pass. |
| Status colors | Status meaning is never conveyed by color alone -- status text labels ("up", "down") and glow animation (pulse for alerts) provide redundant signals |
| Focus indicators | All interactive elements (buttons, inputs, menu items) must show visible `focus-visible` ring using `ring-2 ring-primary` |
| Keyboard navigation | Context menu: Escape closes, arrow keys navigate items. SidePanel: Escape closes. SearchOverlay: Escape clears/closes. All existing keyboard handlers retained. |
| Icon accessibility | Material Symbols icons in context menu and toolbar are decorative (adjacent text label provides meaning). Use `aria-hidden="true"` on icon elements. Toolbar icon-only buttons retain existing `title` attribute for tooltip. |
| Reduced motion | `animate-pulse` on status indicators must respect `prefers-reduced-motion: reduce` -- add `motion-reduce:animate-none` class |

## Performance Constraints

| Constraint | Target | Method |
|-----------|--------|--------|
| Canvas 60fps at 100+ nodes | Maintained | DeviceCard glow uses `box-shadow` only (GPU-composited). No `backdrop-filter` on canvas nodes per D-08. No pseudo-element radial gradients. |
| Material Symbols font size | Under 100KB woff2 | Subset to ~20 required icons using `material-symbols` npm package or manual subsetting |
| Theme switch latency | Under 16ms (1 frame) | CSS variable swap via `data-theme` attribute -- no React re-render needed for color changes |
| DeviceCard memo stability | No unnecessary re-renders | Existing `memo()` comparator retained. New glow classes are derived from existing `status` and `alertStatus` props (no new props needed) |

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser client -> `Client.readPump()` | Untrusted WebSocket control frames enter the backend here |
| Hub subscription registry -> pipeline targeted send | Incorrect registry state would leak detail deltas to the wrong client |
| Disconnect/unregister -> retained client state | Stale subscription state must be cleared when a socket goes away |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-01 | Tampering | `parseClientControlMessage` | mitigate | Validate JSON shape, message type, and UUID format before mutating subscription state; ignore malformed frames instead of applying partial state. Severity: HIGH. |
| T-43-02 | Information Disclosure | `DetailSubscribers` selection | mitigate | Return only clients whose tracked `detailDeviceID` exactly matches the requested device ID; no wildcard or multi-device subscription path exists in this phase. Severity: HIGH. |
| T-43-03 | Denial of Service | `readPump()` malformed-frame handling | mitigate | Keep `maxMessageSize`, log-and-continue on invalid control payloads, and preserve pong deadline handling so a bad frame cannot break all connections. Severity: MEDIUM. |
| T-43-04 | Elevation of Privilege | per-client subscription state | mitigate | One client maps to one active detail device only; `subscribe_detail` replaces prior state instead of accumulating multiple targets. Severity: HIGH. |
| T-43-05 | Repudiation | disconnect cleanup | mitigate | `removeClient()` clears registry membership and safe-closes the socket so stale subscriptions cannot continue silently after disconnect. Severity: HIGH. |

Security gate status: ASVS L1, block_on=high. No HIGH-severity issue remains once parser and hub cleanup tests pass.
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SNMP counter samples -> rate helper | Untrusted counter values and timestamps flow into the pure rate helper |
| Device inventory -> sanity bound | Interface speed metadata from discovery constrains whether rates are considered plausible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-05 | Tampering | Counter reset interpreted as valid traffic | mitigate | Explicit `new < old` detection discards the interval and forces warm-up before the next user-visible rate. Severity: HIGH if unmitigated because the current path can emit absurd values. |
| T-40-06 | Denial of Service | Overspeed sample produces implausible utilization spikes | mitigate | `max(tx, rx) > 110% * speed` discards the interval and resets baseline. Severity: HIGH if unmitigated because the UI can show impossible throughput and trigger false alarms. |
| T-40-07 | Tampering | Long scheduling gaps producing misleading average rates | mitigate | `dt > 3*expectedInterval` is treated as a gap; the sample is discarded and the next interval is warm-up only. Severity: MEDIUM. |
| T-40-08 | Information Disclosure | Additional worker tests exposing internal rate mechanics | accept | Tests live under `internal/worker` and do not widen any runtime interface. Severity: LOW. |

Security gate status: ASVS L1, block_on=high. All HIGH-severity findings are mitigated within this plan.
</threat_model>

Create the Docker infrastructure for the MikroTik Theia development environment: multi-stage backend Dockerfile with CGO, frontend placeholder, docker-compose with profiles and custom networking, and Air hot-reload configuration.

<task type="auto">
  <name>Task 1: Backend Dockerfile with multi-stage CGO build and Air hot-reload</name>
  <files>Dockerfile, .air.toml, .dockerignore</files>
  <action>
Create Dockerfile with multi-stage build:

Stage 1 - "builder":
- Base: golang:1.22-bookworm (Debian-based for CGO compatibility, not Alpine — Alpine has musl libc issues with CGO)
- Install gcc, libc6-dev for CGO compilation
- Set CGO_ENABLED=1
- Copy go.mod, go.sum first (layer caching for dependencies)
- Run go mod download
- Copy source code
- Build: `go build -o /app/theia ./cmd/theia/`

Stage 2 - "dev" (used for development with Air):
- Base: golang:1.22-bookworm
- Install gcc, libc6-dev, air (go install github.com/air-verse/air@latest)
- Set CGO_ENABLED=1
- Set WORKDIR /app
- Copy go.mod, go.sum, run go mod download
- EXPOSE 8080
- CMD ["air", "-c", ".air.toml"]
- Note: source code is volume-mounted in dev mode, NOT copied

Stage 3 - "production":
- Base: debian:bookworm-slim
- Install ca-certificates, libc6 (for CGO runtime)
- Copy --from=builder /app/theia /usr/local/bin/theia
- Create /data directory for SQLite
- EXPOSE 8080
- CMD ["theia"]

Create .air.toml:
- root = "."
- build.cmd = "go build -o ./tmp/theia ./cmd/theia/"
- build.bin = "./tmp/theia"
- build.include_ext = ["go", "yaml"]
- build.exclude_dir = ["tmp", "data", "vendor", ".git"]
- build.delay = 1000 (1 second debounce)
- log.time = true

Create .dockerignore:
- .git
- tmp/
- data/
- *.db
- .planning/
- .claude/
- .agents/
- frontend_example.jpg
  </action>

  </verify>
  <done>Backend Dockerfile builds successfully with CGO enabled. Dev stage includes Air and gcc. Production stage is minimal Debian slim. .air.toml configured for Go hot-reload. .dockerignore excludes non-build files.</done>

<task type="auto">
  <name>Task 2: Docker-compose with profiles, custom network, and frontend placeholder</name>
  <files>docker-compose.yml, Dockerfile.frontend</files>
  <action>
Create Dockerfile.frontend:
- Base: nginx:alpine
- Create a simple index.html inline: `<html><body><h1>MikroTik Theia</h1><p>Frontend placeholder - Phase 2</p></body></html>`
- Copy to /usr/share/nginx/html/index.html
- EXPOSE 80

Create docker-compose.yml with the following structure:

Networks:
- `theia-net`: custom bridge network, IPAM config with subnet 172.20.0.0/24, gateway 172.20.0.1

Volumes:
- `theia-data`: named volume for SQLite database persistence

Services:

1. `backend`:
   - build: context ".", dockerfile "Dockerfile", target "dev"
   - profiles: ["dev", "test"]
   - container_name: theia-backend
   - ports: ["8080:8080"]
   - volumes:
     - ".:/app" (source mount for hot-reload)
     - "theia-data:/app/data" (persistent SQLite)
   - environment:
     - THEIA_DB_PATH=/app/data/theia.db
     - THEIA_LISTEN_ADDR=:8080
     - CGO_ENABLED=1
   - networks:
     theia-net:
       ipv4_address: 172.20.0.2
   - depends_on:
     snmp-router: { condition: service_healthy }
     snmp-switch: { condition: service_healthy }
     snmp-ap: { condition: service_healthy }
   - healthcheck:
     test: ["CMD", "curl", "-f", "http://localhost:8080/api/v1/health"]
     interval: 10s
     timeout: 5s
     retries: 3
     start_period: 30s

2. `frontend`:
   - build: context ".", dockerfile "Dockerfile.frontend"
   - profiles: ["dev"]
   - container_name: theia-frontend
   - ports: ["3000:80"]
   - networks: [theia-net]

3. `snmp-router` (MikroTik router simulator):
   - image: polinux/snmpd:latest (or net-snmp based image)
   - profiles: ["dev", "test"]
   - container_name: theia-snmp-router
   - volumes: ["./docker/snmp/router:/etc/snmp:ro"]
   - ports: ["10161:161/udp"]
   - networks:
     theia-net:
       ipv4_address: 172.20.0.10
   - healthcheck:
     test: ["CMD", "snmpget", "-v2c", "-c", "public", "localhost", "1.3.6.1.2.1.1.1.0"]
     interval: 5s
     timeout: 3s
     retries: 5
     start_period: 10s

4. `snmp-switch` (Cisco switch simulator):
   - Same pattern as snmp-router
   - volumes: ["./docker/snmp/switch:/etc/snmp:ro"]
   - ports: ["10162:161/udp"]
   - networks:
     theia-net:
       ipv4_address: 172.20.0.11
   - healthcheck: same pattern

5. `snmp-ap` (Ubiquiti AP simulator):
   - Same pattern as snmp-router
   - volumes: ["./docker/snmp/ap:/etc/snmp:ro"]
   - ports: ["10163:161/udp"]
   - networks:
     theia-net:
       ipv4_address: 172.20.0.12
   - healthcheck: same pattern

IMPORTANT: For the SNMP simulator image, use `instrumentisto/snmpd` or a simple custom Dockerfile based on debian with net-snmp installed if polinux/snmpd is unavailable. Check Docker Hub availability and fall back to a custom Dockerfile at `docker/snmp/Dockerfile.snmpd` if needed. The Dockerfile should:
- Base: debian:bookworm-slim
- Install snmpd, snmp, libsnmp-dev
- EXPOSE 161/udp
- CMD ["snmpd", "-f", "-Lo", "-C", "-c", "/etc/snmp/snmpd.conf"]

The SNMP config files (snmpd.conf and MIB data) will be created in Plan 02. For now, create empty placeholder directories:
- docker/snmp/router/
- docker/snmp/switch/
- docker/snmp/ap/
with a placeholder snmpd.conf in each that just has:

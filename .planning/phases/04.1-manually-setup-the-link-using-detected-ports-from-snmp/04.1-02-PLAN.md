1. **frontend/src/types/api.ts** — Add `InterfaceInfo` type:
   ```typescript
   export interface InterfaceInfo {
     if_name: string;
     if_descr: string;
     speed: number;
     oper_status: string;
     admin_status: string;
     in_use: boolean;
     in_use_by?: string;
   }
   ```
   Add `parseInterfacesResponse(payload: unknown): InterfaceInfo[]` parser (same pattern as other parsers — extract `data` array, map records).

2. **frontend/src/api/client.ts** — Add four new API functions:
   - `fetchDeviceInterfaces(deviceId: string): Promise<InterfaceInfo[]>` — GET `/api/v1/devices/${id}/interfaces`, parse with `parseInterfacesResponse`
   - `createLink(payload: { source_device_id: string; source_if_name: string; target_device_id: string; target_if_name: string }): Promise<Link>` — POST `/api/v1/links`, parse response data as Link
   - `updateLink(id: string, payload: { source_if_name: string; target_if_name: string }): Promise<Link>` — PUT `/api/v1/links/${id}`, parse response
   - `deleteLink(id: string): Promise<void>` — DELETE `/api/v1/links/${id}`

3. **frontend/src/components/LinkCreatePanel.tsx** — New component:
   - Props: `{ devices: Device[]; onCreated: () => void; onClose: () => void }`
   - State: sourceDeviceId, targetDeviceId, sourceIfName, targetIfName, sourceInterfaces (InterfaceInfo[]), targetInterfaces (InterfaceInfo[]), loading, error
   - When source device changes: fetch interfaces via `fetchDeviceInterfaces(sourceDeviceId)`, populate source port dropdown
   - When target device changes: same for target
   - Device dropdowns: simple `<select>` elements listing all devices by hostname
   - Interface dropdowns render each option as: `{if_name}  {formatSpeed(speed)}  {oper_status}` with disabled styling for DOWN ports (but still selectable per user decision). Options with `in_use: true` show "(in use by {in_use_by})" annotation.
   - Sort interfaces: UP first, then DOWN, alphabetically within each group (matching backend sort)
   - Submit button calls `createLink()`, on success calls `onCreated()` which triggers topology reload
   - Style: dark theme matching existing panels (bg-surface, text-gray-200, border-border colors from Tailwind config)

4. **frontend/src/components/LinkDetailsPanel.tsx** — New component:
   - Props: `{ link: Link; devices: Device[]; onUpdated: () => void; onDeleted: () => void; onClose: () => void }`
   - Display: Shows `{sourceDevice.hostname}:{link.source_if_name} ↔ {targetDevice.hostname}:{link.target_if_name}`, discovery_protocol badge, timestamps
   - Edit mode: toggle button. When editing, show interface dropdowns (same as create panel) pre-populated with current values. Save calls `updateLink()`.
   - Delete button: shows confirmation prompt ("Delete this link?"). On confirm, calls `deleteLink()` then `onDeleted()`.
   - Style consistent with existing InterfaceStatsPanel and DeviceConfigPanel patterns

**Prometheus alert rules:**
Create `docker/prometheus/alert_rules.yml`:
```yaml
groups:
  - name: theia_device_alerts
    rules:
      - alert: DeviceDown
        expr: up{job="snmp"} == 0
        for: 30s
        labels:
          severity: critical
        annotations:
          summary: "Device {{ $labels.instance }} is down"

      - alert: HighCPU
        expr: hrProcessorLoad{job="snmp"} > 85
        for: 60s
        labels:
          severity: warning
        annotations:
          summary: "High CPU on {{ $labels.instance }}"

      - alert: LinkDown
        expr: ifOperStatus{job="snmp"} == 2
        for: 30s
        labels:
          severity: critical
        annotations:
          summary: "Interface {{ $labels.ifDescr }} down on {{ $labels.instance }}"

      - alert: HighLinkUtilization
        expr: (rate(ifHCInOctets{job="snmp"}[5m]) * 8 / (ifHighSpeed{job="snmp"} * 1000000)) > 0.8
        for: 60s
        labels:
          severity: warning
        annotations:
          summary: "High utilization on {{ $labels.ifDescr }} of {{ $labels.instance }}"
```

Adjust the exact PromQL expressions to match the actual metric names exposed by the snmp_exporter in this dev stack. Check `docker/prometheus/prometheus.yml` for the job name and `docker/snmp_exporter/` for available metrics.

**Link alert visuals (ALRT-02, ALRT-03):**

LinkEdge.tsx:
- Add `alertStatus?: AlertStatus` to LinkEdgeData
- When `alertStatus === 'down'`: stroke color becomes red (#ff1744), add a pulsing animation (CSS keyframes or Tailwind animate-pulse), stroke width increases to 3
- When `alertStatus === 'degraded'`: stroke color becomes amber (#ffc107), stroke width 2.5
- Alert styling takes priority over utilization-based coloring

Canvas.tsx:
- In the snapshot merge useEffect, compute link alert status from the alerts array:
  - A link is "down" if there's a firing critical alert matching the link's interface (alertName: "LinkDown", matching the source or target device and interface)
  - A link is "degraded" if there's a firing warning alert (e.g., "HighLinkUtilization") matching the interface
- Create an `alertStatusForLink` helper function similar to `alertStatusForDevice`:
  - Inputs: link (Link), alerts (AlertDTO[])
  - Match alerts by device_id (link.source_device_id or link.target_device_id)
  - For interface-specific alerts, match by checking if alert summary contains the interface name (best-effort matching since Prometheus alert labels vary)
  - Return AlertStatus: 'down' if critical firing alert matches, 'degraded' if warning, 'normal' otherwise
- Pass `alertStatus` to edge data in the snapshot merge

LinkEdge.tsx:

These optimizations ensure React Flow's built-in virtualization works effectively with 100+ nodes by minimizing unnecessary re-renders.

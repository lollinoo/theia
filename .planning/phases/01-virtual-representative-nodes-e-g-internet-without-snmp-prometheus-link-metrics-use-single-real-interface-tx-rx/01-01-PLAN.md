<objective>
Add virtual device type support to the Go backend: domain model, migration, API validation, service probe skip, and poller skip.

Purpose: Enable creation and management of virtual/representative nodes (Internet, Cloud, Server, Generic) that have no SNMP or Prometheus association, forming the foundation for the frontend to render and link these nodes.
Output: Working backend API that accepts virtual device creation, virtual link creation, and correctly skips SNMP probing for virtual devices.
</objective>

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage in Phase 4 |
|-------|-------|-------------------|
| xs | 4px | Dot-to-label gap in pill area buttons, inline icon gaps |
| sm | 8px | Pill internal padding (`py-2 px-2`), area card metric row gaps |
| md | 16px | Area card internal padding, pill section separators, fade mask inset |
| lg | 24px | Hub section spacing (`gap-6`), area card padding (`p-6`), watermark offset from viewport edge |
| xl | 32px | Hub top margin below pill, Hub header bottom padding |
| 2xl | 48px | Hub aggregate stats to area grid gap (`gap-12` = 48px) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Pill top offset from viewport: 16px (`top-4`) -- aligns with existing NavBar z-index position
- Ghost node offset from connected node: 200px -- not a UI spacing value, a graph layout distance
- Touch target minimum: 44px height for all pill buttons (accessibility)

## Interaction Contracts

### NavigationPill

| Interaction | Behavior |
|-------------|----------|
| Click "Global" | Switch to Hub view (`activeView = 'hub'`, `selectedAreaId = null`). "Global" button shows active state |
| Click area button | Switch to Canvas view (`activeView = 'canvas'`, `selectedAreaId = area.id`). Canvas filters to that area. Clicked button shows active state with glowing dot |
| Click Hub icon | Switch to Hub view (same as "Global") |
| Click Devices icon | Switch to Dashboard view (`activeView = 'dashboard'`). Pill simplifies to: Hub icon + "Devices" label + theme toggle |
| Click theme toggle | Toggle `data-theme` between dark/light. Pill surface transitions (glassmorphism / solid) |
| Area overflow scroll | Horizontal scroll with CSS mask fade edges (16px fade on each side). No scroll arrows. `scrollbar-hide` utility |
| View swap | Instant -- no animation (D-08). Pill active state is the sole orientation indicator |

### NavigationPill States

| State | Visual |
|-------|--------|
| Active "Global" | `text-on-bg font-semibold`, text-shadow glow in dark mode (`0 0 8px rgba(245,245,247,0.3)`), solid bg in light mode (`bg-on-bg text-white`) |
| Active area | `text-on-bg font-semibold`, area dot glows (`box-shadow: 0 0 8px {areaColor}`), no text-shadow |
| Inactive area | `text-on-bg-secondary font-regular`, area dot static (no glow) |
| Hover inactive | `text-on-bg` transition 200ms |

### AreaHub

| Interaction | Behavior |
|-------------|----------|
| Click area card | Navigate to that area's filtered canvas (`activeView = 'canvas'`, `selectedAreaId = area.id`). Pill updates to show area as active |
| Hub with no areas | Show aggregate stats (global data) + CTA card prompting area creation in Settings |
| Data loading | Show stat cards with "--" placeholder values; area cards show skeleton or nothing until data arrives |

### Area Card Hover

| Property | Default | Hover | Transition |
|----------|---------|-------|------------|
| Border color | `border-outline` | Area accent color (inline `borderColor`) | 200ms ease |
| Bloom opacity | 0.10 dark / 0.06 light | 0.20 dark / 0.10 light | opacity transition 200ms |
| Cursor | default | pointer | -- |

### Ghost Node

| Interaction | Behavior |
|-------------|----------|
| Click ghost node | Navigate to that device's area (`onGhostClick` calls `onAreaSelect(device.area_id)`) |
| Hover ghost node | Border transitions from `border-outline-subtle` (dashed) to `border-outline` (dashed). Background opacity increases from 0.4 to 0.6 |
| No hover metrics | Ghost nodes show no tooltip, no metrics, no context menu |

### Watermark

| State | Text | Opacity Dark | Opacity Light |
|-------|------|-------------|---------------|
| Hub / Global | "GLOBAL TOPOLOGY" | 0.12 | 0.06 |
| Area filtered | Area name (uppercase) | 0.12 | 0.06 |
| Transition | Fade opacity to 0 over 75ms, swap text, fade opacity back over 75ms (total 150ms per D-19) | -- | -- |

**Source:** CONTEXT.md D-01 through D-25 (all interaction decisions), dark/light HTML mocks (active states verified from code).

## Accessibility Notes

| Concern | Implementation |
|---------|----------------|
| Pill button minimum touch target | 44px height (py-2 + text = ~36px; ensure min-h-[44px] on buttons) |
| Hub icon button aria-label | `aria-label="View Area Hub"` on the Hub icon button in NavigationPill |
| Devices icon button aria-label | `aria-label="View Devices"` on the Devices icon button in NavigationPill |
| Ghost node click target | 120px wide, content height -- sufficient |
| Watermark | `pointer-events-none`, `aria-hidden="true"`, decorative only |
| Area card keyboard navigation | Cards are `<button>` or have `role="button"` + `tabIndex={0}` + `onKeyDown` for Enter/Space |
| Pill area buttons | Standard `<button>` elements with area name as accessible label |
| Theme toggle | `aria-label` with current state description (existing pattern from NavBar) |
| Color-only health indicators | Health label text ("Optimal", "Degraded", "Critical") provides non-color meaning alongside green/yellow/red |
| Reduced motion | Bloom hover transitions respect `motion-reduce:transition-none` |

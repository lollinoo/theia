<objective>
Wire virtual device support through the API handlers and poller layer. HandleCreate validates virtual device requests (device_type, tags, optional IP), HandleBatchAdd passes the new deviceType parameter, link handler relaxes if_name for virtual sides while rejecting both-virtual links, and the poller skips virtual devices during re-probe cycles.

Purpose: Plan 01 built the domain constant and service layer. This plan connects those changes to the HTTP and worker layers, completing the backend virtual device story. After this plan, the API accepts virtual device creation requests and the poller ignores them.
Output: Updated device handler, link handler, and poller with tests.
</objective>

## v1.3.7 Requirements

### Virtual Device Backend

- [x] **VIRT-01**: User can create a virtual node with device_type "virtual", a display name, and a subtype (Internet, Cloud, Server, Generic) — Phase 8, satisfied
- [x] **VIRT-02**: Virtual nodes can exist without an IP address (multiple empty-IP nodes coexist) — Phase 8, satisfied
- [x] **VIRT-03**: Virtual nodes with IP are ping-probed for status (up/down glow) — Phase 8, satisfied (via MetricsCollector probe_success + blackbox exporter)
- [x] **VIRT-04**: Virtual nodes without IP have "unknown" status with no probing — Phase 8, satisfied
- [x] **VIRT-05**: SNMP poller skips virtual devices during re-probe cycle — Phase 8, satisfied

### Virtual Node Rendering

- [x] **VIRT-06**: Virtual node renders as compact card with subtype-specific Material Symbol icon and display name — Phase 9, satisfied
- [x] **VIRT-07**: Virtual node with IP shows StatusDot and IP address line (200px card) — Phase 9, satisfied
- [x] **VIRT-08**: Virtual node without IP shows icon and label only (160px card, no body) — Phase 9, satisfied
- [x] **VIRT-09**: Material Symbols font subset expanded with language, cloud, and dns glyphs — Phase 9, satisfied

### Virtual Node Forms

- [x] **VIRT-10**: AddDevicePanel has Physical Device / Virtual Node toggle at top of form — Phase 10, satisfied
- [x] **VIRT-11**: Virtual form shows subtype radio group, required display name, optional IP — Phase 10, satisfied
- [x] **VIRT-12**: LinkCreatePanel hides interface selector for virtual side — Phase 10, satisfied
- [x] **VIRT-13**: Link creation rejects both devices being virtual — Phase 10, satisfied

### Link Metrics & Context Menu

- [x] **VIRT-14**: Link to virtual node displays real interface tx/rx throughput on edge label — Phase 9, satisfied
- [x] **VIRT-15**: Link bandwidth label shows real interface speed only (no mismatch indicator) — Phase 9, satisfied
- [x] **VIRT-16**: Canvas context menu for virtual nodes omits WebFig and Per-Interface Stats — Phase 10, satisfied

## Out of Scope (unchanged)

| Feature | Reason |
|---------|--------|
| Virtual-to-virtual links | No metrics source on either side; not useful for topology visualization |
| Auto-discovery of virtual endpoints | Virtual nodes are user-defined concepts, not discoverable |
| Custom virtual node icons | Subtype icons (language/cloud/dns/hub) cover the key use cases |
| Virtual node grouping/nesting | Adds complexity; virtual nodes participate in existing area system |
| Ping probing implementation (ICMP) | Delegated to blackbox exporter via Prometheus; no custom ICMP code |

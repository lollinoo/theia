## Trust Boundaries

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-40-01 | Tampering | Future enrichment accidentally overriding SNMP-authoritative fields | mitigate | This plan does not implement any Prometheus query path inside `PerformanceCollector`; enrichment remains the zero-value contract only, so SNMP authority is preserved by construction. Severity: MEDIUM. |
| T-40-02 | Denial of Service | Missing interface speed causing bogus sanity-bound assumptions | mitigate | `InterfaceCounterSnapshot.SpeedBps` is copied from discovered interface metadata and defaults to `0`, never to an invented speed; downstream rate logic will treat missing/zero speed as unknown rather than safe. Severity: MEDIUM. |
| T-40-03 | Information Disclosure | Collector result structs carrying enrichment fields for later plans | accept | The result types are internal-only Go structs inside `internal/collector`; they do not widen any API surface. Severity: LOW. |
| T-40-04 | Tampering | Partial OID absence silently zeroing metrics | mitigate | The collector preserves nil pointer semantics from `snmp.PollDeviceMetrics` and does not coerce missing OIDs to `0`, preventing false healthy readings. Severity: MEDIUM. |

Security gate status: ASVS L1, block_on=high. No HIGH-severity threat remains open in this plan.
</threat_model>

# Security Policy

## Supported Versions

Security fixes are prioritized for the latest published release and the `master`
branch.

## Reporting a Vulnerability

Please do not open a public issue for a security vulnerability.

Use GitHub private vulnerability reporting from the repository's **Security** tab
when it is available. If private vulnerability reporting is unavailable, contact
the maintainer through the repository owner's GitHub profile and avoid sharing
exploit details in public channels.

Include:

- Affected version or commit.
- Impact and affected component.
- Reproduction steps or proof of concept.
- Any known mitigations.

The maintainer will acknowledge valid reports and coordinate disclosure through
GitHub Security Advisories when appropriate.

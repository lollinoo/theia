## Typography

| Role | Size | Weight | Line Height | Usage in This Phase |
|------|------|--------|-------------|---------------------|
| Body | 14px | 400 (regular) | 1.5 | IP address line in virtual card body (`font-mono text-[14px]`) |
| Label | 13px | 600 (semibold) | 1.4 | Display name label in virtual card header (`font-mono text-[13px] font-semibold`) |
| Caption | 11px | 400 (regular) | 1.3 | IP address prefix label ("IP:") in card body |
| Icon | 24px | 400 (normal) | 1.0 | Material Symbols subtype icon via `size={24}` prop |

Source: Aligned with existing DeviceCard.tsx type scale -- physical cards use `text-[15px] font-bold` for hostname, `text-[14px] font-bold font-mono` for IP, `text-[13px] font-medium` for detail text. Virtual cards use slightly smaller text to reinforce their "secondary" visual weight.

Font families:
- Display name: `font-mono` (JetBrains Mono Variable) -- matches D-06 from CONTEXT.md
- IP address: `font-mono` (JetBrains Mono Variable) -- matches existing physical card pattern
- Icon: Material Symbols Rounded at 24px -- matches D-05 (22-24px range, using 24px for clean scaling)

### Status Glow (D-03)

| Status | Glow | Animation |
|--------|------|-----------|

## Component Inventory

### New: VirtualDeviceCard (rendered within DeviceCard.tsx)

Two variants based on IP presence:

#### Variant A: IP-Bearing Virtual Node (200px)

```
+--------------------------------------+  <-- 200px width
|         [SubtypeIcon 24px]           |  <-- centered, text-on-bg-secondary
|        display_name [StatusDot]      |  <-- font-mono 13px semibold, truncate
|--------------------------------------|
|  IP:                    192.168.1.1  |  <-- font-mono 14px, bg-bg section
+--------------------------------------+
```

- Outer wrapper: dashed border (`border border-dashed border-outline-subtle`), `bg-surface/80`, `rounded-[12px]`
- Header: centered layout, `px-3 py-2`, icon centered on own line, name + StatusDot on second line
- Body: `bg-bg px-3 py-2 rounded-b-[12px]`, IP address row with label and value
- Total estimated height: ~90px

#### Variant B: No-IP Virtual Node (160px)

```
+----------------------------+  <-- 160px width
|     [SubtypeIcon 24px]     |  <-- centered, text-on-bg-secondary
|       display_name         |  <-- font-mono 13px semibold, truncate
+----------------------------+
```

- Outer wrapper: dashed border, `bg-surface/80`, `rounded-[12px]`
- Header only: centered layout, `px-3 py-2`, icon centered on own line, name below
- No body section
- Total estimated height: ~56px

### Modified: edgeBuilder.ts -- buildEdgeData()

For links where one device is virtual:

| Property | Physical-Physical Value | Virtual Link Value |
|----------|------------------------|--------------------|
| `bandwidthLabel` | `formatBandwidth(min)` or `formatBandwidth(source) (!)` | `formatBandwidth(realInterfaceSpeed)` -- single side only |
| `speedMismatch` | `true` if speeds differ | Always `false` |
| `throughputLabel` | `up X / down Y` with interface names | `up X / down Y` rates only (no interface name per D-09) |

### Modified: nodeBuilder.ts -- buildTopologyNodes()

- Detect `device.device_type === 'virtual'` to set card width and `isVirtual` flag
- Pass `subtype` from `device.tags?.subtype` into DeviceNodeData

### Extended: DeviceNodeData interface

New fields:

| Field | Type | Default | Purpose |
|-------|------|---------|---------|
| `isVirtual` | `boolean` | `false` | Enables virtual card rendering branch |
| `subtype` | `string` | `undefined` | Material Symbol icon name lookup |

### Extended: DeviceType union

From: `'router' | 'switch' | 'ap' | 'unknown'`
To: `'router' | 'switch' | 'ap' | 'unknown' | 'virtual'`

### Subtype Icon Mapping (D-12)

## Interaction Contract

### Virtual Node Interactions

| Interaction | Behavior | Matches Physical? |
|-------------|----------|-------------------|
| Drag | Fully draggable, position saved | Yes |
| Select (click) | Selection ring via primary color wrapper | Yes |
| Context menu (right-click) | Opens context menu (Phase 10 will filter irrelevant items) | Yes (partial -- menu filtering is Phase 10) |
| Hover | Glow effect on wrapper (area-colored if in area, primary if no area) | Yes |
| Edit mode handles | Shows 4 connection handles (top/right/bottom/left) on hover | Yes |
| Ghost representation | When filtered by area, virtual nodes outside the area appear as ghost (120px, dashed, semi-transparent) | Yes (uses existing isGhost path) |

| Property | Physical Node | Virtual Node | Ghost Node |
|----------|--------------|--------------|------------|
| Width | 260px | 200px (IP) / 160px (no IP) | 120px |

| Opacity | 1.0 | 1.0 (fully opaque) | 0.4 (`bg-surface/40`) |

| Draggable | Yes | Yes | No |

### Current Subset (21 glyphs)

The existing `material-symbols-rounded-subset.woff2` (26KB) contains glyphs used across the application. The exact list is managed by the build process.

### Required Additions (D-09, D-12)

| `cloud` | U+E2BD | Cloud subtype icon |

### Build Process

Regenerate the woff2 subset file using `pyftsubset` (fonttools) or Google Fonts API with the updated glyph list. The font file lives at `frontend/public/fonts/material-symbols-rounded-subset.woff2`. The `@font-face` declaration in `index.css` does not need changes -- only the font file itself is rebuilt.

## CSS Specifications

### Virtual Card Dashed Border

```css
/* Tailwind classes */
border border-dashed border-outline-subtle

/* Resolves to */
border-width: 1px;
border-style: dashed;
border-color: var(--nt-outline-subtle); /* #2A2A2E dark / #E0E0E5 light */
```

### Virtual Card Muted Background

```css
/* Tailwind class */
bg-surface/80

/* Resolves to */
background-color: rgb(from var(--nt-surface) r g b / 0.8);
/* Dark: rgba(30, 30, 33, 0.8) */
/* Light: rgba(237, 237, 240, 0.8) */
```

This provides the "visually secondary" feel from D-01 while keeping the card fully readable (not semi-transparent like ghost nodes at /40).

### Virtual Card Border Radius

```css
/* Same as physical cards */
border-radius: 12px; /* rounded-[12px] */
```

### Virtual Card Wrapper (selection/status glow)

Same wrapper pattern as physical DeviceCard:
- Wrapper `div` with `rounded-[13.5px]` (12px card + 1.5px padding)
- `background` set to area color, `var(--color-primary)` when selected, or `var(--color-outline)` default
- `padding: 1.5px` default, `2px` when selected/highlighted/down/degraded
- Status glow classes identical to physical cards

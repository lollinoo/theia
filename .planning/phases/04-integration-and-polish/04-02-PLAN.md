**InterfaceStatsPanel.tsx:** Create the panel content component for per-interface statistics.

Props:
- `link: Link` -- the link being inspected
- `sourceDevice: Device` -- source endpoint device
- `targetDevice: Device` -- target endpoint device
- `snapshot: SnapshotPayload | null` -- current WebSocket snapshot for live data

Rendering:
- Two sections, one per interface endpoint (source and target)
- Each section shows:
  - Device hostname + interface name as header
  - Interface speed (formatted with `formatBandwidth` from LinkEdge.tsx) and admin/oper status
  - Live TX/RX throughput from snapshot.link_metrics, formatted with `formatThroughput`
  - Utilization percentage with color coding (reuse `utilizationColor` from metrics.ts)
- If snapshot is null or metrics not available for an interface, show "--" placeholders
- Dark themed, consistent with app styling (bg-bg-elevated sections, text-text-primary/secondary)
- Layout: stacked vertically, each interface in a card-like section with subtle border

Data extraction logic:
- For source interface: find in `snapshot.link_metrics[link.source_device_id]` where `if_name` matches `link.source_if_name`
- For target interface: find in `snapshot.link_metrics[link.target_device_id]` where `if_name` matches `link.target_if_name`
- Interface metadata (speed, admin/oper status): find in `sourceDevice.interfaces` / `targetDevice.interfaces` by matching `if_name`

**Canvas.tsx:** Wire InterfaceStatsPanel into the SidePanel:
- When `panelContent.type === 'interfaceStats'` and data has a `link`, render `<InterfaceStatsPanel>` inside `<SidePanel>`
- Pass the current `snapshot` from useWebSocket so the panel gets live updates automatically
- Side panel title: "Interface Stats" or the link's source/target device names

## Integration Findings

### Gap 1 — Hardcoded bridge port `:1337` in frontend (medium severity)

**Affects:** TRAY-04, BRIDGE-05, WINBOX-01, WINBOX-02

Phase 29 enables users to configure `ListenPort` in `~/.config/winbox-bridge/config.json`. However, the frontend hardcodes `localhost:1337` in three places:

- `frontend/src/hooks/useBridgeHealth.ts:3` — `BRIDGE_HEALTH_URL = 'http://localhost:1337/health'`
- `frontend/src/components/Canvas.tsx:256` — launch POST to `http://localhost:1337/launch`
- `frontend/src/components/Dashboard.tsx:115` — launch POST to `http://localhost:1337/launch`

When a user configures a non-default port, the bridge binds there but the frontend polls/posts to `:1337`, reporting bridge as "not running" and silently failing all WinBox launches.

**Fix options:**
1. Add `bridge_port` to Theia settings (analogous to `bridge_secret`) — user configures once in Settings UI
2. Document that `ListenPort` must remain `1337` for Theia integration

### Gap 2 — `testSSHProfile` dead code in client.ts (low severity)

**Affects:** CRED-05 (minor)

`frontend/src/api/client.ts:415-426` exports `testSSHProfile` calling `/api/v1/ssh-profiles/{id}/test` — route does not exist. Function is never imported or called.

**Fix:** Rename to `testCredentialProfile` with path `/api/v1/credential-profiles/${id}/test`, or delete.

1. Create `frontend/src/components/MaterialIcon.tsx`:

```typescript
interface MaterialIconProps {
  name: string;
  className?: string;
  size?: number;
}

export function MaterialIcon({ name, className = '', size = 18 }: MaterialIconProps) {
  return (
    <span
      className={`material-symbols-rounded ${className}`}
      aria-hidden="true"
      style={size !== 18 ? { fontSize: size } : undefined}
    >
      {name}
    </span>
  );
}
```

2. Create `frontend/src/components/MaterialIcon.test.tsx`:

Write tests using vitest + @testing-library/react following existing test patterns (see DeviceCard.test.tsx for import style). Use `render` and `screen` from testing-library. Test:
- Renders span with text content matching `name` prop (e.g., "settings")
- Span has class `material-symbols-rounded`
- Span has `aria-hidden="true"`
- When `className="text-primary"` is passed, span has both `material-symbols-rounded` and `text-primary` classes
- When `size={24}` is passed, span has `style` with `fontSize: 24`
- When default size (18) is used, span does NOT have a style attribute for fontSize

Per D-01 (Material Symbols for all icons), D-02 (Rounded style). The component enforces consistent `aria-hidden`, sizing, and font-variation-settings via the CSS class.

  <action>
    Create `cmd/winbox-bridge/main.go` — a single-file Go binary (package main). Use only stdlib (`net/http`, `os`, `os/exec`, `flag`, `encoding/json`, `log`, `os/signal`, `runtime`, `path/filepath`). No external dependencies. No CGO.

    **Imports:** stdlib only — `encoding/json`, `flag`, `fmt`, `log`, `net/http`, `os`, `os/exec`, `os/signal`, `path/filepath`, `runtime`, `strings`, `syscall`.

    **CLI flags (per D-02, D-04):**
    - `--theia-origin` string, default `"http://localhost:3000"` — the accepted Origin header value
    - `--winbox-path` string, default `""` — manual override for WinBox executable path
    - `--listen` string, default `":1337"` — listen address (for testability, but documented default is 1337)

    **WinBox Discovery function `discoverWinBox() string` (per D-01, D-03):**
    - If `--winbox-path` flag is non-empty: validate that the file exists and is executable. If valid, return that path. If invalid, log a warning and return "" (bridge will start but /launch returns 503).
    - Otherwise search PATH using `exec.LookPath`:
      - On Windows (`runtime.GOOS == "windows"`): try `exec.LookPath("winbox64.exe")`, then `exec.LookPath("winbox.exe")`
      - On Linux/macOS: try `exec.LookPath("winbox")`
    - If PATH search fails, check platform defaults:
      - Windows: `C:\Program Files\WinBox\winbox64.exe` (check `os.Stat`)
      - Linux: `/usr/bin/winbox` (check `os.Stat`)
      - macOS: `/Applications/WinBox.app/Contents/MacOS/WinBox` (check `os.Stat`)
    - Return the discovered path, or `""` if nothing found. Log the result.

    **Security middleware `securityCheck(allowedOrigin string, next http.Handler) http.Handler` (per D-04, D-05, D-06, D-07):**
    - Extract `Origin` header from request. If empty or does not exactly match `allowedOrigin`, return 403 `{"error":"forbidden"}`.
    - Extract `Host` header from request. If not exactly `"localhost:1337"`, return 403 `{"error":"forbidden"}`.
    - If both pass, call `next.ServeHTTP(w, r)`.
    - This middleware wraps ALL routes — /health, /launch, and the CORS preflight handler.

    **CORS handling (Claude's discretion):**
    - The browser sends `fetch()` from `http://localhost:3000` to `http://localhost:1337` — this IS cross-origin, so the browser WILL send a preflight OPTIONS request.
    - The security middleware runs first. For OPTIONS requests that pass Origin+Host validation:
      - Set `Access-Control-Allow-Origin` to the `--theia-origin` value (NOT `*`)
      - Set `Access-Control-Allow-Methods` to `"GET, POST, OPTIONS"`
      - Set `Access-Control-Allow-Headers` to `"Content-Type"`
      - Return 204 No Content
    - For non-OPTIONS requests that pass validation, also set the `Access-Control-Allow-Origin` response header so the browser allows the response to be read.

    **Route handler: `handleHealth(w http.ResponseWriter, r *http.Request)` (per D-12):**
    - Only accept GET method. Return 405 `{"error":"method not allowed"}` for others.
    - Return 200 with body `{"ok":true}` and Content-Type `application/json`.

    **Route handler: `handleLaunch(winboxPath string) http.HandlerFunc` (per D-08, D-09, D-10, D-11):**
    - Only accept POST method. Return 405 `{"error":"method not allowed"}` for others.
    - Parse JSON body into struct: `type launchRequest struct { IP string \x60json:"ip"\x60; Username string \x60json:"username"\x60; Password string \x60json:"password"\x60 }` — exactly 3 fields, NO executable/command field (D-09).
    - Validate: if IP, Username, or Password is empty string, return 400 `{"error":"ip, username, and password are required"}`.
    - If `winboxPath == ""`, return 503 `{"error":"winbox executable not found — use --winbox-path to specify location"}` (per D-03).
    - Construct command args: `[]string{winboxPath, req.IP, req.Username, req.Password}` (per D-08).
    - Use `os.StartProcess` with `&os.ProcAttr{Files: []*os.File{os.Stdin, nil, nil}}` to detach stdout/stderr. Alternatively use `exec.Command(winboxPath, ip, username, password)` with `cmd.Stdout = nil; cmd.Stderr = nil; cmd.Start()` — then immediately call `cmd.Process.Release()` to detach (per D-10).
    - If Start fails, return 500 `{"error":"failed to launch WinBox"}` (per D-11). Do NOT include the internal error message in the response.
    - On success, return 200 `{"ok":true}`.

    **main() function (per D-16):**
    - Parse flags: `--theia-origin`, `--winbox-path`, `--listen`.
    - Call `discoverWinBox()` to find WinBox path (skipped if `--winbox-path` set).
    - Log startup info: `log.Printf("winbox-bridge starting on %s (origin=%s winbox=%s)", listenAddr, theiaOrigin, winboxPath)`. If winboxPath is empty, log: `log.Println("WARNING: WinBox executable not found — /launch will return 503. Use --winbox-path to specify.")`.
    - Create `http.ServeMux`, register `/health` and `/launch` routes.
    - Wrap mux with `securityCheck(theiaOrigin, mux)`.
    - Create `http.Server{Addr: listenAddr, Handler: securityWrapped}`.
    - Set up graceful shutdown on SIGINT/SIGTERM (same pattern as cmd/theia/main.go).
    - Call `server.ListenAndServe()`.

    **Helper: `writeJSON(w http.ResponseWriter, status int, v interface{})` — sets Content-Type, writes status, encodes JSON.**
    **Helper: `writeError(w http.ResponseWriter, status int, msg string)` — calls writeJSON with `map[string]string{"error": msg}`.**

    Create `cmd/winbox-bridge/main_test.go` — test all behaviors listed above. Use `httptest.NewServer` or `httptest.NewRecorder` with the handler chain. For process launch tests, inject the process start function as a variable so tests can mock it:

    **Testability pattern:** Define a package-level variable `var startProcess = defaultStartProcess` where `defaultStartProcess` calls `exec.Command(...).Start()`. Tests override `startProcess` to a mock that records the call and returns nil or an error. Reset in `t.Cleanup`.

    The test file should use build tag `//go:build !integration` or no build tag (standard unit tests). Tests must NOT actually launch WinBox.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to Bridge | Cross-origin fetch from Theia frontend (localhost:3000) to bridge (localhost:1337). Untrusted — any webpage could attempt this. |
| Bridge to OS | Bridge spawns an OS process (WinBox). Must be restricted to only the WinBox executable. |
| Local network | Any process on the same machine can reach localhost:1337. Origin+Host validation is the primary defense. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Spoofing | /launch endpoint | mitigate | Validate Origin header matches --theia-origin exactly (D-04). Return 403 if mismatch. Prevents arbitrary websites from triggering WinBox launch. |
| T-26-02 | Spoofing | All endpoints | mitigate | Validate Host header is exactly "localhost:1337" (D-05). Prevents DNS rebinding attacks where attacker's domain resolves to 127.0.0.1. |
| T-26-03 | Tampering | /launch request body | mitigate | launchRequest struct has exactly 3 fields (ip, username, password). No executable/command field exists in the struct — Go JSON unmarshalling ignores unknown fields. Hardcoded executable path from discovery (D-09). |
| T-26-04 | Information Disclosure | /launch error response | mitigate | On StartProcess failure, return generic "failed to launch WinBox" message (D-11). Internal error details logged server-side only. |
| T-26-05 | Information Disclosure | Credential in CLI args | accept | WinBox requires username+password as CLI arguments. This is visible in process listings (`ps aux`). Accepted because: (1) bridge runs on user's own machine, (2) WinBox itself requires this invocation pattern, (3) no alternative method exists. |
| T-26-06 | Elevation of Privilege | Arbitrary command execution | mitigate | Bridge discovers WinBox at startup and stores the path in a variable. Request body cannot influence which executable is launched. No code path exists to pass an executable name from HTTP request to os.StartProcess (D-09). |
| T-26-07 | Denial of Service | Rapid /launch requests | accept | Bridge is a local tool for a single user. Rate limiting not needed — worst case spawns many WinBox windows, which is self-correcting (user closes them). |
| T-26-08 | Tampering | Command injection via IP/username/password fields | mitigate | Fields are passed as separate arguments to exec.Command (not concatenated into a shell string). exec.Command does NOT invoke a shell — each arg is a separate argv element, preventing injection. |
</threat_model>

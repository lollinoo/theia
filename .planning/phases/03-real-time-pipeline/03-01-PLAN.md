<task type="auto" tdd="true">
  <name>Task 2: Metrics domain types and Prometheus Go client</name>
  <files>internal/domain/metrics.go, internal/metrics/prometheus.go, internal/metrics/prometheus_test.go, go.mod</files>
  <behavior>
    - Test: NewPromClient with a mock HTTP server returning Prometheus API JSON parses DeviceMetrics correctly (CPU, memory, uptime, temperature)
    - Test: QueryDeviceMetrics returns "--" equivalent zero values when Prometheus returns empty result
    - Test: QueryLinkMetrics parses ifHCInOctets/ifHCOutOctets rate values into TX/RX bps
    - Test: QueryAlerts parses Prometheus /api/v1/alerts response into AlertState structs
  </behavior>
  <action>
  1. Create `internal/domain/metrics.go` with domain types:
     ```go
     type DeviceMetrics struct {
         DeviceID    uuid.UUID
         CPUPercent  *float64  // nil = no data
         MemPercent  *float64  // nil = no data
         TempCelsius *float64  // nil = not available (show N/A)
         UptimeSecs  *float64  // nil = no data
         CollectedAt time.Time
     }

     type LinkMetrics struct {
         LinkID       string
         DeviceID     uuid.UUID
         IfName       string
         TxBps        *float64  // nil = no data
         RxBps        *float64  // nil = no data
         Utilization  *float64  // max(tx,rx) / speed, 0-1
         CollectedAt  time.Time
     }

     type AlertState struct {
         DeviceID  uuid.UUID
         Severity  string    // "warning", "critical"
         AlertName string
         State     string    // "firing", "resolved"
         Summary   string
     }

     type AlertStatus string
     const (
         AlertStatusNormal   AlertStatus = "normal"
         AlertStatusDegraded AlertStatus = "degraded"  // warning severity
         AlertStatusDown     AlertStatus = "down"       // critical severity or device unreachable
     )
     ```

  2. Create `internal/metrics/prometheus.go` with `PromClient`:
     - Constructor: `NewPromClient(baseURL string, httpClient *http.Client) *PromClient`
     - Method: `QueryDeviceMetrics(ctx context.Context, deviceIPs []string) (map[string]DeviceMetrics, error)` -- queries Prometheus for CPU (hrProcessorLoad), memory (hrStorageUsed/hrStorageSize), uptime (sysUpTime), temperature. Uses `/api/v1/query` endpoint with PromQL. Key by device IP (instance label).
     - Method: `QueryLinkMetrics(ctx context.Context, deviceIPs []string) (map[string][]LinkMetrics, error)` -- queries rate(ifHCInOctets[5m]) and rate(ifHCOutOctets[5m]) for interface throughput. Key by device IP.
     - Method: `QueryAlerts(ctx context.Context) ([]AlertState, error)` -- queries `/api/v1/alerts` for firing alerts.
     - Use standard `net/http` for Prometheus HTTP API calls. Parse JSON responses manually (no Prometheus client library needed -- the HTTP API is simple).
     - PromQL queries to use:
       - CPU: `avg by (instance) (hrProcessorLoad{instance=~"$targets"})`
       - Memory: `100 * hrStorageUsed{hrStorageDescr="Physical memory"} / hrStorageSize{hrStorageDescr="Physical memory"}`
       - Uptime: `sysUpTime{instance=~"$targets"} / 100` (sysUpTime is in hundredths of seconds)
       - Temperature: `entPhySensorValue{entPhySensorType="8",instance=~"$targets"}` (type 8 = celsius)
       - Interface TX rate: `rate(ifHCOutOctets{instance=~"$targets"}[5m]) * 8` (bytes to bits)
       - Interface RX rate: `rate(ifHCInOctets{instance=~"$targets"}[5m]) * 8`

  3. Create `internal/metrics/prometheus_test.go` with httptest.Server mocking Prometheus responses. Test happy path parsing and empty/error responses.

  No external Go dependencies needed -- use net/http + encoding/json only.

At the start of `handleSubmit`, before the existing physical validation, add an early-return branch for virtual mode:
```typescript
async function handleSubmit(e: React.FormEvent) {
  e.preventDefault();
  if (isVirtual) {
    if (!displayName.trim()) {
      setError('Display Name is required.');
      return;
    }
    setLoading(true);
    setError(null);
    try {
      await createDevice({
        hostname: displayName.trim(),
        ip: virtualIp.trim() || undefined,
        device_type: 'virtual',
        tags: {
          display_name: displayName.trim(),
          virtual_subtype: virtualSubtype,
        },
        area_ids: areaIds.length > 0 ? areaIds : undefined,
      });
      onDeviceAdded();
    } catch (err) {
      setError(err instanceof Error ? err.message : 'Failed to add virtual node.');
    } finally {
      setLoading(false);
    }
    return;
  }
  // ... existing physical submit logic unchanged ...
}
```

Note: For the virtual path, `snmp` is intentionally omitted from the payload (now optional in the interface). The backend skips SNMP validation entirely when `device_type` is `'virtual'` (Phase 8 D-08), so no placeholder needed. RESEARCH Pitfall 2 suggested `snmp: { version: '2c' }` as a placeholder but that was written before the decision to make `snmp` optional in the interface -- omitting it is cleaner and the backend handles it correctly.

`hostname` is set to `displayName.trim()` so the backend has a value. `ip` is sent as undefined when empty (backend allows empty IP for virtual).

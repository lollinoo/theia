    **Create(ctx context.Context) (*domain.InstanceBackup, error):**

    1. Generate backup UUID: `backupID := uuid.New()`
    2. Create directory: `{backupDir}/{backupID}/`
    3. Build filename per D-07: `theia-backup-{YYYYMMDD}-{HHMMSS}-v{version.Version}.tar.gz` using `time.Now().UTC().Format("20060102-150405")`
    4. Create initial DB record with status `InstanceBackupStatusRunning`
    5. Call `backupDatabase(ctx, tempDBPath)` — writes VACUUM'd copy to `{backupDir}/{backupID}/theia.db.tmp`
    6. Compute SHA-256 of the database copy (streaming with `sha256.New()` + `io.Copy`)
    7. Read migration version from backup DB: open backup DB read-only, query `SELECT version FROM schema_migrations`
    8. Build manifest struct with all fields per D-08:
       - `Version: 1`
       - `AppVersion: version.Version`
       - `GitCommit: version.GitCommit`
       - `MigrationVersion`: from step 7
       - `CreatedAt`: ISO 8601 (`time.Now().UTC().Format(time.RFC3339)`)
       - `DBSHA256`: from step 6
       - `BackupFileCount`: count of device backup files found
       - `TotalSizeBytes`: sum of all archived file sizes
       - `EncryptionKeyHash`: `computeEncryptionKeyHash(s.encryptionKey)` per D-09
    9. Create archive at temp path `{backupDir}/{backupID}/{filename}.tmp`:
       - Open file, wrap in `gzip.NewWriter`, wrap in `tar.NewWriter`
       - Add `manifest.json` (JSON-marshal manifest, write as tar entry)
       - Add `theia.db` (stream from temp DB file)
       - Walk `s.deviceBackupDir` if it exists, add files under `backups/` prefix (use `filepath.Rel` for relative paths). Skip `s.backupDir` to avoid circular inclusion per Pitfall 6
       - Add `known_hosts` if `s.knownHostsPath` exists (use basename `known_hosts` as tar entry name)
       - Close tar writer, then gzip writer, then file
    10. Rename temp archive to final path: `os.Rename(tempPath, finalPath)`
    11. Compute archive SHA-256 (streaming) and write sidecar file per D-11: `{finalPath}.sha256` with content `{hash}  {basename}\n`
    12. Update DB record: status = `InstanceBackupStatusSuccess`, set `FileName`, `FilePath`, `SizeBytes`, `SHA256`, `AppVersion`, `MigrationVersion`
    13. Clean up temp DB file
    14. On any error: update DB record to `InstanceBackupStatusFailed` with error message, clean up temp files, return error

    **backupDatabase(ctx context.Context, destPath string) error:**
    Per D-01, D-02, D-03:
    1. `PRAGMA wal_checkpoint(TRUNCATE)` — log warning if fails, do not abort
    2. `VACUUM INTO ?` with `destPath` as parameter — if parameterized fails, use `fmt.Sprintf("VACUUM INTO '%s'", destPath)` with path validation (reject `'` and `;` characters)
    3. Open backup DB read-only (`destPath + "?mode=ro"`)
    4. Run `PRAGMA integrity_check`, scan result, verify equals `"ok"`
    5. Close backup DB (defer)

    **computeEncryptionKeyHash(key []byte) string:**
    Per D-09: `sha256.Sum256(key[:8])`, return `hex.EncodeToString(hash[:])`

    **readMigrationVersion(dbPath string) (int, error):**
    Open DB read-only, query `SELECT version FROM schema_migrations`, close, return version.

    **addFileToTar(tw *tar.Writer, name string, sourcePath string) error:**
    Open file, stat for size, write tar header (Name, Size, Mode 0644, ModTime), io.Copy file data.

    **addBytesToTar(tw *tar.Writer, name string, data []byte, modTime time.Time) error:**
    Write tar header with len(data) as size, write data bytes.

    **computeFileHash(path string) (string, error):**
    Open file, create `sha256.New()`, `io.Copy(hasher, file)`, return `hex.EncodeToString(hasher.Sum(nil))`.

    **List(ctx context.Context) ([]domain.InstanceBackup, error):**
    Delegate to `s.repo.List()`.

    **GetByID(ctx context.Context, id uuid.UUID) (*domain.InstanceBackup, error):**
    Delegate to `s.repo.GetByID(id)`.

    **Delete(ctx context.Context, id uuid.UUID) error:**
    1. Get backup by ID from repo
    2. If found and FilePath is not empty: `os.RemoveAll(filepath.Dir(backup.FilePath))` to remove the UUID subdirectory
    3. Call `s.repo.Delete(id)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| VACUUM INTO destination path | Service constructs file path for database copy — must not allow path traversal |
| Tar archive entry names | Files added to archive with relative paths — must not use absolute paths |
| Device backup directory walk | Service walks a configurable directory — must not traverse outside intended scope |
| Encryption key in memory | Service holds encryption key bytes for manifest hash computation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Tampering | instance_backup_service.go backupDatabase() | mitigate | VACUUM INTO destination path is constructed by the service (UUID subdir + fixed filename), never from user input. Path validated with `filepath.Clean`. Parameterized query used for VACUUM INTO. |
| T-15-05 | Tampering | instance_backup_service.go Create() tar entries | mitigate | Tar entry names constructed with `filepath.Rel()` producing relative paths. No absolute paths in archive. Entries starting with `/` or `..` are rejected. |
| T-15-06 | Information Disclosure | instance_backup_service.go manifest | mitigate | Manifest contains only checksums, version numbers, and timestamps. Encryption key hash uses SHA-256 of first 8 bytes only (per D-09) — not the full key. No credentials, passwords, or raw secrets in manifest. |
| T-15-07 | Information Disclosure | instance_backup_service.go temp files | mitigate | Temp DB copy and temp archive cleaned up in defer blocks. Failed backups have temp files removed. Service does not leave `.tmp` files after completion. |
| T-15-08 | Denial of Service | instance_backup_service.go Create() | accept | VACUUM INTO can briefly increase SQLite read lock contention. Acceptable: backup is infrequent (manual or scheduled), and WAL mode minimizes lock duration. No concurrent backup guard needed for v1.4.0. |
| T-15-09 | Information Disclosure | instance_backup_service.go archive on disk | accept | Archive contains the full database (with AES-256-GCM encrypted credential fields). Archive security relies on filesystem permissions (0644) and encryption key secrecy. No additional archive-level encryption for v1.4.0. |
| T-15-10 | Tampering | instance_backup_service.go circular inclusion | mitigate | When walking deviceBackupDir for archive inclusion, explicitly skip instanceBackupDir to prevent including previous instance backups in new archives (exponential growth). |
</threat_model>

**frontend/src/types/api.ts:** Define TypeScript types matching the backend JSON:API format:
```typescript
export type DeviceType = 'router' | 'switch' | 'ap' | 'unknown';
export type DeviceStatus = 'up' | 'down' | 'probing' | 'unknown';

export interface DeviceInterface {
  id: string;
  if_index: number;
  if_name: string;
  if_descr: string;
  speed: number; // bits per second
  admin_status: string;
  oper_status: string;
}

export interface Device {
  id: string;
  hostname: string;
  ip: string;
  device_type: DeviceType;
  status: DeviceStatus;
  sys_name: string;
  hardware_model: string;
  managed: boolean;
  interfaces: DeviceInterface[];
}

export interface Link {
  id: string;
  source_device_id: string;
  source_if_name: string;
  target_device_id: string;
  target_if_name: string;
  discovery_protocol: string;
}
```

Also define helper functions to parse JSON:API responses into these types (extract from `data[].attributes` and `data[].relationships.interfaces.data`).

**frontend/src/api/client.ts:** API client module with:
- `fetchDevices(): Promise<Device[]>` — GET /api/v1/devices, parse JSON:API response
- `fetchLinks(): Promise<Link[]>` — GET /api/v1/links, parse response

Use plain `fetch()` (no axios dependency). Base URL defaults to '' (relative, goes through Vite proxy). Handle errors by throwing with meaningful messages. Parse the JSON:API format: for devices, extract id from data[].id, flatten attributes, extract interfaces from relationships.

- Selected state: accent border (#00d4ff)
- Size: compact, roughly 200px wide. Don't make it too tall.
- React Flow handles: one on left (target), one on right (source) for link connections. Style handles to be small/subtle.

IMPORTANT: Define the nodeTypes object OUTSIDE the component (at module level) to prevent React Flow re-renders. Example:

Style:
- Line: stroke color #4a4a5e (border-subtle), stroke-width 2
- Label: text-xs, bg-canvas background with padding, text-secondary color
- Use straight or smoothstep edge path (smoothstep preferred for cleaner look)

Define edgeTypes at module level (same pattern as nodeTypes).

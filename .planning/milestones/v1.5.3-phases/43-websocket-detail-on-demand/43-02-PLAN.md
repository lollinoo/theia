<objective>
Build the backend detail-delivery path: after a normal device poll completes,
the pipeline emits a targeted `snapshot_delta` for subscribed clients using the
shared snapshot contract and without increasing overview broadcast load.

Purpose:
- Preserve D-06 and D-07 by changing delivery timing only, not scheduler
  cadence or polling frequency.
- Preserve D-08 and D-09 by shipping only backend-computed device-level detail
  metadata in this phase.
- Reuse the Phase 42 pipeline/state seams instead of inventing a side channel.

Output:
- `snapshot_builder.go` helper for single-device detail deltas on the shared
  contract.
- `pipeline.go` targeted post-poll send path keyed by hub subscription state.
- Runtime tests proving subscribed delivery, non-leakage, and no overview
  cadence regression.
</objective>

}

- `broadcastOnce()` remains the overview path from Phase 42.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| State store -> targeted detail payload | Internal runtime state becomes client-visible detail metadata here |
| Hub subscription registry -> `publishSubscribedDetailDelta()` | Wrong recipient selection would leak one device's detail to the wrong socket |
| Poll completion -> detail send timing | This seam must not change scheduler cadence or trigger extra polls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-06 | Information Disclosure | `publishSubscribedDetailDelta` recipient selection | mitigate | Resolve recipients only through `DetailSubscribers(device.ID)` and assert unsubscribed clients receive nothing in runtime tests. Severity: HIGH. |
| T-43-07 | Tampering | `buildDeviceDetailDelta` payload shape | mitigate | Reuse `DeviceMetricsDTO` optional fields on the shared snapshot contract and keep link metrics/alerts/models out of targeted detail payloads in tests. Severity: HIGH. |
| T-43-08 | Denial of Service | subscription-driven cadence | mitigate | `publishSubscribedDetailDelta()` runs only after normal `runTask()` completion and never calls scheduler mutation, `broadcastOnce()`, or extra poll logic. Severity: HIGH. |
| T-43-09 | Information Disclosure | per-interface detail scope | mitigate | Targeted detail delta intentionally omits `link_metrics`, preventing Phase 43 from widening into faster WS-delivered per-interface detail. Severity: HIGH. |
| T-43-10 | Repudiation | detail hashing compatibility | mitigate | Include additive detail fields in device-metric hash formatting so future delta paths cannot silently miss a detail change. Severity: MEDIUM. |

Security gate status: ASVS L1, block_on=high. No HIGH-severity issue remains once targeted-send and non-leakage tests pass.
</threat_model>

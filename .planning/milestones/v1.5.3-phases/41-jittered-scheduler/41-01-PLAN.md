    ```go
    package scheduler

    import (
        "hash/fnv"
        "math"
        "math/rand"
        "time"

        "github.com/google/uuid"
    )

    func initialOffset(deviceID uuid.UUID, interval time.Duration) time.Duration
    func jitteredNext(lastFire time.Time, interval time.Duration, rnd *rand.Rand) time.Time
    ```

    Implement exactly like this:
    - `initialOffset`:
      1. return `0` when `interval <= 0`
      2. hash `deviceID[:]` with `fnv.New64a()`
      3. compute `fraction := float64(sum) / float64(math.MaxUint64)`
      4. return `time.Duration(fraction * float64(interval))`
      5. if floating-point rounding returns `interval`, clamp to `interval - time.Nanosecond`
    - `jitteredNext`:
      1. return `lastFire` when `interval <= 0`
      2. return `lastFire.Add(interval)` when `rnd == nil`
      3. compute `jitterSpan := float64(interval) * 0.1`
      4. sample a delta from `[-jitterSpan, +jitterSpan]`
      5. return `lastFire.Add(interval + sampledDelta)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Device UUID -> initial offset | Persistent device identity influences scheduling spread and must not create burst clustering |
| Device DB row -> performance override | DB-stored `poll_interval_override` changes the performance cadence used by the scheduler |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-01 | Denial of Service | initial offset helper | mitigate | `initialOffset` is deterministic and bounded to `[0, interval)`; distribution tests catch burst clustering regressions before runtime code lands. Severity: HIGH if unmitigated because all devices could align on one cycle boundary. |
| T-41-02 | Tampering | poll interval override handling | mitigate | `EffectiveInterval` ignores nil/zero/negative overrides and only applies positive overrides to the performance class, preventing malformed rows from zeroing the schedule. Severity: MEDIUM. |
| T-41-03 | Repudiation | volatility priority ordering | mitigate | Priority is encoded in a pure helper with explicit tests, so later queue logic cannot silently invert the intended `performance > operational > static` order. Severity: MEDIUM. |
| T-41-04 | Information Disclosure | PollTask device snapshot | accept | `PollTask` is an internal Go struct under `internal/scheduler`; no API or log surface is widened by carrying the device snapshot. Severity: LOW. |

Security gate status: ASVS L1, block_on=high. No HIGH-severity issue remains open after the deterministic spread tests are added.
</threat_model>

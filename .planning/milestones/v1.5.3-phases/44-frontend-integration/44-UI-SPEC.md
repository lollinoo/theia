## Visual Hierarchy

- The card header remains the first read: vendor icon, hostname, then the primary status group at the right edge.
- The primary status group is one compact unit: existing glow dot plus an explicit health label (`Healthy`, `Warning`, `Critical`, `Unknown`) in 12px semibold text.
- The identity/model row remains intact below the header. Phase 44 adds status metadata without replacing hostname, model, or IP context.
- Freshness and polling cadence occupy one compact metadata row in the body. Freshness reads first, cadence reads second.
- The metrics strip remains the lowest-priority block. Health and freshness are not embedded into the CPU/MEM/TEMP/UP grid.

---

## Interaction Contract

- Health ownership is backend-only. The frontend must render the backend `health` enum directly and must not infer health from CPU, memory, temperature, alert severity, or reachability.
- Freshness is a separate signal derived from `last_polled_at` plus `expected_poll_interval_seconds`. Use three tiers only: `Fresh` when age is `<= 2x` expected interval, `Stale` when age is `> 2x` and `<= 5x`, `Dead` when age is `> 5x` or there is still no successful poll timestamp.
- Freshness copy format is fixed: `{Tier} · {relative age}`. Examples: `Fresh · 12s ago`, `Stale · 2m ago`, `Dead · 9m ago`. If the device has never completed a poll, use `Dead · Waiting for first poll`.
- Polling cadence copy on the canvas card is fixed operator language: `Polling every 30s`, `Polling every 1m`, `Polling every 5m`. Do not show raw backend labels like `core`, `standard`, or `low` on the card.
- The device configuration panel keeps the override-first workflow. Show a read-only context line first: `Default cadence: every 30s (core class)` or equivalent time-first copy, then the editable override controls beneath it.
- Polling override editing remains inline within the polling section. Do not introduce a modal, secondary drawer, or full-panel submit button just for polling. Success feedback is the existing inline `Saved` text.
- The editable actions are only `Use device default`, one of the preset seconds values, or a custom seconds override. Direct class switching is out of scope for Phase 44.
- Card-level health, freshness, and cadence metadata must render from the shared overview snapshot state on the map itself. Opening the side panel is not required to expose these values.
- Phase 43 detail-subscription ownership remains unchanged: the canvas device panel is still the only owner of `subscribe_detail` / `unsubscribe_detail`. Phase 44 adds presentation, not a second state channel.
- Scope stays on the canvas and device configuration panel. Dashboard/table parity and broader card redesign are deferred.

## Copywriting Contract

|---------|------|
| Primary CTA | `Save polling override` — use only if implementation needs an explicit button; otherwise preserve inline autosave and keep `Saved` as the success confirmation |

### Required Inline Labels

- Health labels: `Healthy`, `Warning`, `Critical`, `Unknown`
- Freshness labels: `Fresh`, `Stale`, `Dead`
- Polling section heading: `Polling Override`
- Polling default context: `Default cadence: every {duration} ({class} class)`
- Polling default option: `Use device default`
- Custom override placeholder: `Seconds (5-3600)`
- Success feedback: `Saved`

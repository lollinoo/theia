  <name>Task 1: Vite WebSocket proxy and Canvas integration</name>
  <files>frontend/vite.config.ts, frontend/src/components/Canvas.tsx</files>
  <action>
  1. Update `frontend/vite.config.ts`:
     - Add WebSocket proxy to the existing API proxy configuration
     - The proxy for `/api/v1/ws` needs `ws: true` to support WebSocket upgrade
     - Ensure the existing `/api` proxy also has `ws: true` or add a separate entry:
       ```typescript
       '/api/v1/ws': {
         target: 'http://backend:8080',
         ws: true,
       }
       ```

  2. Update `frontend/src/components/Canvas.tsx`:
     - Import: `useWebSocket` from hooks, `ReconnectBanner`, metric helper functions from types/metrics
     - Add to Canvas component: `const { snapshot, reconnecting } = useWebSocket('/api/v1/ws');`

     - Create a `useEffect` that fires when `snapshot` changes:
       a. If snapshot is null, skip
       b. Update nodes: for each node, merge `snapshot.device_metrics[node.id]` into `node.data.metrics` and compute `alertStatusForDevice(node.id, snapshot.alerts)` into `node.data.alertStatus`
       c. Update edges: for each edge that has a `link`, find matching link metrics by looking up the link's source_device_id and source_if_name in `snapshot.link_metrics`. Compute:
          - `throughputLabel`: format TX/RX from matching LinkMetricsDTO using `formatThroughput` -- "TX: 1.2M / RX: 456K"
          - `utilization`: from matching LinkMetricsDTO's utilization field
       d. Use `setNodes` and `setEdges` updater functions (not full replace) to merge metric data while preserving positions, selection, etc.
       e. Use `startTransition` to avoid blocking UI during metric updates

     - Staleness handling:
       a. Track last snapshot timestamp in a ref
       b. Set up an interval (every 10s) that checks if `Date.now() - lastSnapshotTime > 2 * pollingInterval`
       c. If stale, clear metrics from all nodes (set metrics to null, alertStatus to undefined)
       d. Default pollingInterval to 60000ms (60s). Could fetch from settings API but simpler to hardcode for now.
       e. Clear the staleness interval on unmount

     - Add `<ReconnectBanner visible={reconnecting} />` inside the Canvas return JSX, before the ReactFlow component

     - Important: Do NOT re-fetch topology data on each snapshot. The existing `loadTopology()` fetches devices/links/positions once on mount. WebSocket snapshots only update metrics overlaid on existing nodes/edges.

     - Edge case: if a device_id in the snapshot doesn't match any node (device added/removed since load), ignore it. Consider adding a periodic re-fetch of topology (e.g., every 5 minutes) but that's optional for this plan.
  </action>

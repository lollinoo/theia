  <action>
Use `github.com/gosnmp/gosnmp` as the SNMP library (widely used, supports v2c and v3).

Create internal/snmp/client.go:
- `type Client struct` wrapping gosnmp.GoSNMP
- `func NewClient(target string, creds domain.SNMPCredentials, timeout time.Duration, retries int) (*Client, error)` — configures gosnmp based on credential version
- `func (c *Client) Connect() error`
- `func (c *Client) Close() error`
- `func (c *Client) Get(oids []string) ([]gosnmp.SnmpPDU, error)` — wraps gosnmp.Get
- `func (c *Client) BulkWalk(oid string) ([]gosnmp.SnmpPDU, error)` — wraps gosnmp.BulkWalk for table walks
- Default timeout: 5 seconds, retries: 2 (configurable)

Create internal/snmp/detector.go:
- `func DetectDeviceType(sysObjectID string, sysDescr string) domain.DeviceType`
- Check sysObjectID first (most reliable):
  - MikroTik: OID prefix 1.3.6.1.4.1.14988 — then check sysDescr for "RouterOS" (Router) vs "SwOS" (Switch)
  - Cisco: OID prefix 1.3.6.1.4.1.9 — check sysDescr for "C9" / "ISR" / "ASR" (Router) vs "Catalyst" / "C2960" / "C3750" (Switch)
  - Ubiquiti: OID prefix 1.3.6.1.4.1.41112 — check sysDescr for "EdgeRouter" (Router) vs "US-" / "UniFi Switch" (Switch) vs "UAP" / "U6" (AP)
- Fallback: check sysDescr for keywords "router", "switch", "access point", "wireless"
- Default: Unknown

Create internal/snmp/discovery.go:
- `type DiscoveryResult struct`: SysName, SysDescr, SysObjectID, HardwareModel string; DeviceType domain.DeviceType; Interfaces []domain.Interface; Neighbors []NeighborInfo
- `type NeighborInfo struct`: RemoteChassisID, RemotePortID, RemoteSysName, LocalIfName, Protocol string (LLDP or CDP)
- `func DiscoverDevice(client *Client) (*DiscoveryResult, error)`:
  1. GET sysName (.1.3.6.1.2.1.1.5.0), sysDescr (.1.3.6.1.2.1.1.1.0), sysObjectID (.1.3.6.1.2.1.1.2.0)
  2. BulkWalk ifTable (.1.3.6.1.2.1.2.2) for all interfaces — extract ifIndex, ifDescr, ifType, ifSpeed, ifAdminStatus, ifOperStatus
  3. BulkWalk ifXTable (.1.3.6.1.2.1.31.1.1) for ifName and ifHighSpeed (64-bit counters)
  4. BulkWalk LLDP-MIB (.1.0.8802.1.1.2.1.4) for LLDP neighbors — parse lldpRemChassisId, lldpRemPortId, lldpRemSysName
  5. BulkWalk CISCO-CDP-MIB (.1.3.6.1.4.1.9.9.23.1.2.1) for CDP neighbors (if LLDP walk returns empty or as supplement)
  6. Call DetectDeviceType with sysObjectID and sysDescr
  7. Parse hardware model from sysDescr (vendor-specific patterns)
- Handle errors per-section: if interface walk fails, still return sysInfo; if neighbor walk fails, still return interfaces

Tests use table-driven tests with mock SNMP PDU data (no real SNMP device needed). Create a mock/helper that returns pre-built PDU slices for each OID tree. Test the parsing logic, not the network calls.

    IMPORTANT: `BackupService.credentialProfileRepo` field is typed as `domain.CredentialProfileRepository` (the interface), which does NOT have `GetWinboxAssignment`. The new assignment methods are on the concrete `*sqlite.CredentialProfileRepo` only. Two options:

    **Preferred approach**: Accept the concrete repo as a parameter (or hold it as a separate field). Since `CredentialProfileHandler` already holds `*sqlite.CredentialProfileRepo` directly, the cleanest approach is to add `GetWinboxCredentials` as a standalone method that takes the needed dependencies:

    Actually, the simplest approach consistent with the codebase: add the method to `BackupService` but pass the concrete repo through. The handler already has both `*service.BackupService` and `*sqlite.CredentialProfileRepo`, so the handler can call:
    ```go
    assignment, err := h.credentialProfileRepo.GetWinboxAssignment(deviceID)
    ```
    then get the device IP from the device repo, then call `BackupService.DecryptSecret` (promote it).

    **Final decision**: Promote `decryptSecret` to `DecryptSecret` (exported) on BackupService. The handler will:
    1. Call `credentialProfileRepo.GetWinboxAssignment(deviceID)` to get username + encrypted_secret
    2. Call `deviceRepo.GetByID(deviceID)` to get IP (need DeviceRepository on the handler -- but adding a dep is more invasive)

    **Simplest approach**: Add a `GetWinboxCredentials` method on `*DeviceCredentialProfileHandler` itself that orchestrates the calls, since the handler has access to both the repo (concrete) and the service (with decryptSecret). To avoid exposing decryptSecret, add a thin method on BackupService:

    ```go
    // GetWinboxCredentials retrieves the decrypted WinBox credentials for a device.
    // Returns the device IP, profile username, and decrypted password.
    func (s *BackupService) GetWinboxCredentials(deviceID uuid.UUID, winboxRow *WinboxAssignmentRow) (password string, err error) {
        return s.decryptSecret(winboxRow.EncryptedSecret)
    }
    ```

    Wait -- that still doesn't get the device IP. Let BackupService hold a `deviceRepo` (it already does!). So:

    ```go
    func (s *BackupService) GetWinboxCredentials(deviceID uuid.UUID, encryptedSecret, username string) (ip, decryptedPassword string, err error) {
        device, err := s.deviceRepo.GetByID(deviceID)
        if err != nil {
            return "", "", fmt.Errorf("device not found: %w", err)
        }
        password, err := s.decryptSecret(encryptedSecret)
        if err != nil {
            return "", "", fmt.Errorf("decrypting credentials: %w", err)
        }
        if password == "" {
            return "", "", fmt.Errorf("WinBox profile has no password configured")
        }
        return device.IP, password, nil
    }
    ```

    The handler calls repo.GetWinboxAssignment, then passes the encrypted secret to BackupService.GetWinboxCredentials. This keeps crypto in the service layer.

       - Extract device ID from path same as above
       - Decode body `{"profile_id": "uuid"}`
       - Parse profile_id as UUID, return 400 if invalid
       - Call `h.credentialProfileRepo.AssignProfile(deviceID, profileID)`
       - If error contains "UNIQUE" -> 409 `"profile already assigned to this device"`
       - If error contains "FOREIGN KEY" -> 404 `"device or profile not found"`
       - Return 201 with `{"data": {"device_id": "...", "profile_id": "..."}}`

    3. `HandleUnassign(w, r)` -- per D-07:
       - Extract device ID AND profile ID from path. Path: `/api/v1/devices/{deviceId}/credential-profiles/{profileId}`
       - Parse: `suffix := strings.TrimPrefix(r.URL.Path, "/api/v1/devices/")`, split on `/`, parts[0]=deviceID, parts[1]="credential-profiles", parts[2]=profileID
       - Call `h.credentialProfileRepo.UnassignProfile(deviceID, profileID)`
       - If error contains "not assigned" -> 404
       - Return 204

    4. `HandleSetWinbox(w, r)` -- per D-08:
       - Extract device ID: `strings.TrimSuffix(r.URL.Path, "/winbox-profile")`, then `extractIDFromPath(trimmed, "/api/v1/devices/")`
       - Decode body `{"profile_id": "uuid"}`
       - Call `h.credentialProfileRepo.SetWinboxProfile(deviceID, profileID)`
       - If error contains "not assigned" -> 404 `"profile is not assigned to this device"`
       - Return 200 with `{"data": {"device_id": "...", "profile_id": "...", "is_winbox": true}}`

    5. `HandleClearWinbox(w, r)` -- per D-09:
       - Extract device ID same as SetWinbox
       - Call `h.credentialProfileRepo.ClearWinboxProfile(deviceID)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /devices/{id}/credential-profiles | Untrusted UUID in path and profile_id in body |
| client -> /devices/{id}/winbox-credentials | Returns decrypted password -- highest sensitivity endpoint |
| handler -> BackupService | Decryption happens in service layer, never in handler |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-05 | Information Disclosure | HandleGetWinboxCredentials | mitigate | Decrypted password ONLY returned by this single endpoint; handler never holds encryption key; password decrypted in BackupService.GetWinboxCredentials |
| T-24-06 | Information Disclosure | HandleListAssignments | mitigate | Response struct explicitly omits encrypted_secret; only returns name/username/port/role/is_winbox |
| T-24-07 | Tampering | HandleAssign body | mitigate | profile_id parsed via uuid.Parse -- rejects non-UUID input; FK constraints prevent assignment to nonexistent device/profile |
| T-24-08 | Tampering | HandleUnassign path | mitigate | Both deviceID and profileID extracted and parsed via uuid.Parse from URL path segments; invalid UUIDs return 400 |
| T-24-09 | Denial of Service | HandleSetWinbox transaction | mitigate | Transaction scope minimal (2 UPDATE statements); SQLite busy_timeout=5000ms configured at connection level |
| T-24-10 | Information Disclosure | WinBox password empty | mitigate | GetWinboxCredentials returns error when decrypted password is empty string -- prevents silent auth failure |
</threat_model>

# Milestone v1.3.7: Virtual/Representative Nodes

**Status:** SHIPPED 2026-04-02
**Phases:** 8-10
**Total Plans:** 6

## Overview

Virtual/representative nodes as first-class entities in the network topology. Users can create virtual devices (Internet, Cloud, Server, Generic subtypes), link them to physical devices, and see them rendered on the canvas with appropriate cards, edge labels, and context menus.

## Phases

### Phase 8: Virtual Device Backend

**Goal**: Backend fully supports virtual devices as a first-class device type with appropriate probe behavior
**Depends on**: None (v1.3.0 codebase)
**Plans**: 2 plans
**Requirements:** [VIRT-01, VIRT-02, VIRT-03, VIRT-04, VIRT-05]

Plans:
- [x] 08-01-PLAN.md — DeviceTypeVirtual constant, partial unique IP migration, AddDevice/probeDevice virtual skip
- [x] 08-02-PLAN.md — Virtual device API handlers (HandleCreate validation, link if_name relaxation, poller skip)

**Details:**
- Added `DeviceTypeVirtual` constant to domain model
- Migration 000009: partial unique index on `devices(ip) WHERE ip != ''` for empty-IP coexistence
- AddDevice accepts `deviceType` parameter (12-arg signature), skips SNMP probe for virtual
- probeDevice virtual guard short-circuits before SNMP/Prometheus logic
- Link handler validates virtual-side if_name relaxation and rejects both-virtual links
- Poller pollAllDevices skips `DeviceTypeVirtual` as defense-in-depth

### Phase 9: Virtual Node Rendering

**Goal**: Virtual devices render as compact, visually distinct cards on the canvas with correct link edge labels
**Depends on**: Phase 8
**Plans**: 2 plans
**Requirements:** [VIRT-06, VIRT-07, VIRT-08, VIRT-09, VIRT-14, VIRT-15]

Plans:
- [x] 09-01-PLAN.md — Virtual card rendering branch, subtype Material Symbol icons, font subset expansion
- [x] 09-02-PLAN.md — Virtual link edge labels with single-side bandwidth and mismatch suppression

**Details:**
- Virtual card early-return branch in DeviceCardInner with dashed border and centered layout
- subtypeIconMap: internet→language, cloud→cloud, server→dns, generic→hub
- IP-bearing virtual cards: 200px with StatusDot and IP body; no-IP: 160px icon+label only
- Material Symbols font subset regenerated with 24 icons (added language, cloud, dns)
- buildEdgeData isVirtualLink early-return: single-side bandwidth, speedMismatch=false
- findLinkMetrics target-device fallback for virtual-source links

### Phase 10: Virtual Node Forms and Context Menu

**Goal**: Users can create virtual nodes and link them to physical devices through adapted frontend forms, with context menus showing only relevant actions for virtual nodes
**Depends on**: Phase 8, Phase 9
**Plans**: 2 plans
**Requirements:** [VIRT-10, VIRT-11, VIRT-12, VIRT-13, VIRT-16]

Plans:
- [x] 10-01-PLAN.md — AddDevicePanel virtual mode toggle, subtype icon cards, and payload extension
- [x] 10-02-PLAN.md — LinkCreatePanel virtual adaptation and Canvas context menu filtering

**Details:**
- Dual-mode AddDevicePanel with Physical Device / Virtual Node segmented toggle
- Virtual form: Display Name (required), 2x2 subtype icon cards, optional IP, shared area multi-select
- CreateDevicePayload extended with optional device_type, ip, snmp fields
- LinkCreatePanel hides interface selector for virtual side, blocks both-virtual with inline error
- Canvas context menu filters to Grafana + Configure for virtual nodes using stable id-based filtering

## Milestone Summary

**Key Decisions:**
- Removed IP-required validation from service layer; handler validates conditionally per device type
- Virtual devices start with status "unknown"; MetricsCollector resolves via probe_success for IP-bearing virtuals
- Virtual card uses early-return branch pattern (matching ghost node pattern)
- Explicit isVirtualLink guard rather than relying on accidental zero-speed behavior
- Stable id-based context menu filtering instead of fragile label string matching

**Issues Resolved:**
- Multiple virtual devices with empty IP coexistence (partial unique index migration)
- SearchableDeviceSelect display for virtual devices without IP (empty-IP guard)
- setSshProfileId casing bug in handleModeSwitch (auto-fixed during execution)

**Issues Deferred:**
- None

**Technical Debt Incurred:**
- ContextMenuItem shared interface lacks 'id' field; Canvas.tsx extends locally
- MetricsCollector SNMP exclusion of virtual devices is implicit via MetricsSource default
- Nyquist VALIDATION.md files remain in draft state for phases 8-10

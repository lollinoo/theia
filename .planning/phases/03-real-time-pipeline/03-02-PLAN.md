  <action>
  1. Add `github.com/gorilla/websocket` dependency: `go get github.com/gorilla/websocket`

  2. Create `internal/ws/messages.go` -- typed message protocol:
     ```go
     // Message is the envelope for all WebSocket messages
     type Message struct {
         Type    string      `json:"type"`    // "snapshot", "metrics", "link_metrics", "alert"
         Payload interface{} `json:"payload"`
     }

     // SnapshotPayload is sent on connect -- full state
     type SnapshotPayload struct {
         DeviceMetrics map[string]DeviceMetricsDTO `json:"device_metrics"` // key: device ID
         LinkMetrics   map[string][]LinkMetricsDTO `json:"link_metrics"`   // key: device ID
         Alerts        []AlertDTO                  `json:"alerts"`
     }

     // DeviceMetricsDTO is the JSON shape sent to frontend
     type DeviceMetricsDTO struct {
         DeviceID    string   `json:"device_id"`
         CPUPercent  *float64 `json:"cpu_percent"`
         MemPercent  *float64 `json:"mem_percent"`
         TempCelsius *float64 `json:"temp_celsius"`
         UptimeSecs  *float64 `json:"uptime_secs"`
         CollectedAt string   `json:"collected_at"` // ISO 8601
     }

     // LinkMetricsDTO is the JSON shape for link throughput
     type LinkMetricsDTO struct {
         DeviceID    string   `json:"device_id"`
         IfName      string   `json:"if_name"`
         TxBps       *float64 `json:"tx_bps"`
         RxBps       *float64 `json:"rx_bps"`
         Utilization *float64 `json:"utilization"` // 0-1
         CollectedAt string   `json:"collected_at"`
     }

     // AlertDTO is the JSON shape for alert state
     type AlertDTO struct {
         DeviceID  string `json:"device_id"`
         Severity  string `json:"severity"`  // "warning", "critical"
         AlertName string `json:"alert_name"`
         State     string `json:"state"`     // "firing", "resolved"
         Summary   string `json:"summary"`
     }
     ```
     Include conversion functions: `DeviceMetricsToDTOs(map[string]DeviceMetrics) map[string]DeviceMetricsDTO` etc.

  3. Create `internal/ws/hub.go` -- connection manager:
     - `Hub` struct with: `clients map[*Client]bool`, `broadcast chan []byte`, `register/unregister chan *Client`, `mu sync.RWMutex`
     - `Client` struct with: `conn *websocket.Conn`, `send chan []byte`
     - `NewHub() *Hub`
     - `Run()` method: goroutine that handles register/unregister/broadcast
     - `Broadcast(msg Message)`: JSON-encodes message, sends to broadcast channel
     - `SendTo(client *Client, msg Message)`: sends to a single client (for initial snapshot)
     - Client write pump: reads from send channel, writes to websocket with write deadline
     - Client read pump: reads from websocket (for ping/pong keepalive), unregisters on close
     - Use 60s write deadline, 60s pong wait, 54s ping period

  4. Create `internal/ws/handler.go` -- HTTP upgrade:
     - `Handler` struct holding `*Hub` and a `snapshotFunc func() *SnapshotPayload`
     - `ServeHTTP(w, r)`: upgrades HTTP to WebSocket, registers client with hub, sends initial snapshot via snapshotFunc, starts read/write pumps
     - Upgrader with CheckOrigin allowing all origins (dev mode)
  </action>

  <name>Task 2: Metrics collector and main.go wiring</name>
  <files>internal/worker/metrics_collector.go, internal/api/router.go, cmd/theia/main.go</files>
  <action>
  1. Create `internal/worker/metrics_collector.go`:
     - `MetricsCollector` struct holding: `promClient *metrics.PromClient`, `hub *ws.Hub`, `deviceRepo domain.DeviceRepository`, `linkRepo domain.LinkRepository`, `settingsRepo domain.SettingsRepository`, `lastSnapshot *ws.SnapshotPayload` (protected by mutex)
     - `NewMetricsCollector(promClient, hub, deviceRepo, linkRepo, settingsRepo) *MetricsCollector`
     - `Start(ctx context.Context)`: background goroutine that:
       a. Reads polling interval from settingsRepo each cycle
       b. Fetches all devices from deviceRepo to get their IPs
       c. Calls promClient.QueryDeviceMetrics, QueryLinkMetrics, QueryAlerts
       d. For link metrics: matches interface names to link records from linkRepo to compute utilization (txBps/ifSpeed)
       e. Converts domain types to DTOs
       f. Stores as lastSnapshot (mutex-protected)
       g. Broadcasts full snapshot via hub.Broadcast with type "snapshot"
       h. Logs errors but does not crash -- missing Prometheus data results in nil metric values
     - `GetSnapshot() *ws.SnapshotPayload`: returns lastSnapshot (for WebSocket connect handler)
     - `Stop()`: cancels context, waits for goroutine

  2. Update `internal/api/router.go`:
     - Add `wsHandler *ws.Handler` parameter to NewRouter

     - Actually, register the WS handler on the mux but ensure the JSONContentType middleware skips WebSocket upgrade requests. The simplest approach: register `/api/v1/ws` with the ws handler directly on the mux, and the upgrade happens before any response writing, so JSON middleware won't interfere. Test this.

     - Pass wsHandler to NewRouter (update signature)

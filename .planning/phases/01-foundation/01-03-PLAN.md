Create internal/service/device_service.go:
- `type DeviceService struct` — holds DeviceRepository, LinkRepository, SettingsRepository, and an SNMP client factory function (for testability)
- `func NewDeviceService(deviceRepo, linkRepo, settingsRepo, snmpClientFactory)`
- `func (s *DeviceService) AddDevice(ctx, ip, hostname string, creds domain.SNMPCredentials, tags map[string]string) (*domain.Device, error)`:
  1. Create device in repo with Status=Probing, Managed=true
  2. Launch goroutine calling s.probeDevice(device)
  3. Return device immediately (async pattern per decisions)
- `func (s *DeviceService) probeDevice(device *domain.Device)`:
  1. Create SNMP client from credentials
  2. Call snmp.DiscoverDevice
  3. Update device fields (sysName, sysDescr, sysObjectID, hardware model, device type, status=Up)
  4. Save interfaces to repo
  5. For each neighbor: check if IP exists in repo → if not, create placeholder (managed=false, status=Unknown)
  6. Create/upsert links between device and neighbors
  7. On failure: set status=Down, log error
- `func (s *DeviceService) UpdateDevice(ctx, id, updates) error` — partial update
- `func (s *DeviceService) DeleteDevice(ctx, id) error` — cascading delete (device + interfaces + links)
- `func (s *DeviceService) GetDevice(ctx, id) (*domain.Device, error)`
- `func (s *DeviceService) GetAllDevices(ctx) ([]domain.Device, error)`
- `func (s *DeviceService) ProbeDevice(ctx, id) error` — re-probe existing device

Use interfaces for SNMP client (pass factory function) so tests can inject mocks without real SNMP.

- JSON content-type middleware (sets Content-Type: application/json on responses)

Create internal/api/router.go:
- `func NewRouter(deviceService *service.DeviceService, settingsRepo domain.SettingsRepository, poller *worker.Poller) http.Handler`
- Uses standard net/http (no framework — Go stdlib is sufficient for this scale)
- Routes:
  - POST   /api/v1/devices         → create device
  - GET    /api/v1/devices         → list all devices
  - GET    /api/v1/devices/{id}    → get device by ID
  - PUT    /api/v1/devices/{id}    → update device
  - DELETE /api/v1/devices/{id}    → delete device
  - POST   /api/v1/devices/{id}/probe → trigger re-probe
  - POST   /api/v1/devices/batch   → batch add devices (returns batch ID)
  - GET    /api/v1/links           → list all links
  - GET    /api/v1/settings        → get all settings
  - PUT    /api/v1/settings/{key}  → update setting
  - GET    /api/v1/health          → health check

Create internal/api/device_handler.go:
- JSON:API response format per decisions: `{"data": {"type": "device", "id": "...", "attributes": {...}, "relationships": {...}}}`
- For list endpoints: `{"data": [...]}`
- Error format: `{"error": "message", "details": ...}` with appropriate HTTP status codes
- POST /devices accepts: `{"ip": "...", "hostname": "...", "snmp": {"version": "2c", "community": "..."}, "tags": {...}}`
- POST /devices/batch accepts: `{"devices": [...]}`  returns `{"batch_id": "...", "status": "processing"}`
- Validate required fields (IP is required, SNMP credentials required)

Create internal/api/settings_handler.go:
- GET /settings returns all settings as key-value pairs
- PUT /settings/{key} accepts `{"value": "..."}` and updates the setting

Create internal/api/health_handler.go:
- Returns `{"status": "ok"|"degraded", "components": {"db": "ok"|"error", "snmp_poller": "running"|"stopped"}}`
- Checks DB with a simple ping query
- Checks poller status from poller reference

Create internal/worker/poller.go:
- `type Poller struct` — runs background goroutine re-probing all managed devices on interval
- `func NewPoller(deviceService, settingsRepo) *Poller`
- `func (p *Poller) Start(ctx context.Context)` — reads polling interval from settings, loops: get all managed devices, probe each with configurable worker pool (semaphore pattern)
- `func (p *Poller) Stop()` — graceful shutdown via context cancellation
- `func (p *Poller) Status() string` — returns "running" or "stopped"
- Worker pool size from settings (default 5)
- Re-reads polling interval from settings each cycle (changes take effect without restart)

## Spacing Scale

All values are Tailwind utility classes mapped to the 4-point base grid. The project uses Tailwind spacing utilities (`p-2` = 8px, `p-3` = 12px, `p-4` = 16px, etc.) rather than named semantic tokens. The effective spacing contract for this phase:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, tight inline padding on icon buttons |
| sm | 8px (`gap-2`, `p-2`) | Gap between row action icons, badge internal padding |
| md | 12px (`p-3`, `px-3`) | Card/row internal padding ‚Äî see exception note below |
| base | 16px (`p-4`) | Panel section padding, form field spacing |
| lg | 24px (`space-y-6`, `gap-6`) | Section breaks within DeviceConfigPanel |
| xl | 32px | Layout-level spacing ‚Äî not used in panel context |

Exceptions:
- **12px exception (deliberate, not a standard token):** `md = 12px` (`p-3`, `px-3`) is outside the standard 4-point multiples set {4, 8, 16, 24, 32, 48, 64}. It is retained because `p-3` is already the established row-padding pattern in `SSHProfileManager.tsx` (line 416 `p-3`, line 18 `px-3 py-2`). Changing it would create visual inconsistency with the existing panel. This token is a Tailwind default (`0.75rem`) and is used exclusively for card/row internal padding to match the pre-existing codebase pattern ‚Äî it is NOT added to the standard spacing scale.
- Assignment list rows use `py-2 px-3` (vertical 8px, horizontal 12px) ‚Äî consistent with the profile card rows in `SSHProfileManager.tsx`
- Key/WinBox icon toggle button: minimum touch target 32px √ó 32px via `p-1` on a 24px icon (matches existing icon button pattern in `SSHProfileManager.tsx` edit/delete buttons)
- "+ Add" button: `px-2 py-1` (8px √ó 4px) matches the existing "New" button in `SSHProfileManager.tsx`

Source: Verified from `SSHProfileManager.tsx` line 18 (`px-3 py-2`), line 396 (`px-2 py-1`), line 416 (`p-3`).

## Typography

All sizes use the Outfit Variable font (`--font-sans`). Monospace fields (private key textarea) use JetBrains Mono (`--font-mono`).

| Role | Size | Weight | Line Height | Tailwind Classes |
|------|------|--------|-------------|------------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 10px | 500 (medium) | 1.2 | `text-xs font-medium uppercase tracking-widest` |
| Name (profile/row primary) | 14px | 500 (medium) | 1.4 | `text-sm font-medium` |
| Secondary / metadata | 12px | 400 (regular) | 1.4 | `text-xs` |

Exactly 2 weights used: 400 (regular) and 500 (medium). No 600/700 weights used in panel contexts.

Source: Verified from `SSHProfileManager.tsx` lines 17‚Äì19 (`labelClass`), line 421 (`text-sm font-medium`), line 423 (`text-xs`).

Role badge (pill):
- Size: 10px (`text-xs`)
- Weight: 500 (`font-medium`)
- Padding: `px-2 py-0.5`
- Background: `bg-surface-high` with `text-on-bg-secondary`
- Radius: `rounded-full`

This is Claude's discretion: pill treatment chosen over plain text because it visually separates the role label from the profile name without additional color complexity.

## Color

The project uses a dual-theme CSS custom property system. All Tailwind classes reference semantic tokens; no raw hex values appear in component code.

| Role | Dark Theme Value | Light Theme Value | Tailwind Token | Usage |
|------|-----------------|-------------------|----------------|-------|
| Dominant (60%) | `#161618` | `#F5F5F7` | `bg-bg` | App canvas background |
| Secondary (30%) | `#1E1E21` / `#262629` | `#EDEDF0` / `#E4E4E8` | `bg-surface` / `bg-surface-high` | Panels, cards, assignment list rows |
| Accent (10%) | `#00E676` (dark) / `#00994D` (light) | same | `text-primary`, `border-primary`, `bg-primary/15` | See reserved list below |
| Destructive | `#FF1744` (dark) / `#C62828` (light) | same | `text-status-down`, `bg-status-down`, `border-status-down/30`, `bg-status-down/10` | Delete confirmation UI only |

Accent (`primary`) reserved for:
1. The designated WinBox key icon ‚Äî filled/active state (`text-primary`)
2. Active auth method toggle button border + background (`border-primary bg-primary/15 text-primary`) ‚Äî inherited from `SSHProfileManager` form
3. Save/submit button (`bg-primary`) in CredentialProfileManager form
4. Input focus ring (`focus:border-primary focus:ring-1 focus:ring-primary/30`) on all form fields
5. Required field asterisk (`text-status-down`) ‚Äî uses destructive, not primary

Non-designated WinBox key icon: `text-on-bg-secondary` (muted, not accent).

Source: Verified from `SSHProfileManager.tsx` lines 194‚Äì215 (auth method toggle), line 265 (save button `bg-primary`), line 17‚Äì18 (inputClass focus ring).

## Interaction States

### Credentials Section (DeviceConfigPanel)

| State | Visual |
|-------|--------|
| Loading (assignments fetching) | Single line: `text-xs text-on-bg-secondary` "Loading credentials..." ‚Äî no skeleton, matches `SSHProfileManager` loading pattern |
| Empty (no assignments) | Placeholder text: "No credentials assigned. Add a profile to enable WinBox launch." ‚Äî `text-xs text-on-bg-secondary` (source: CONTEXT.md D-07) |
| Populated | Rows in `rounded-lg bg-surface-high p-3` cards, each showing: name (`text-sm font-medium`), role pill, key toggle, remove button |
| Adding | `<select>` dropdown appears in-panel with available unassigned profiles; selecting immediately calls `assignCredentialProfile` API; on success refreshes assignment list |
| WinBox designation toggle ‚Äî designating | Click on non-designated key icon ‚Üí calls `PUT /api/v1/devices/{id}/winbox-profile` ‚Üí refreshes assignment list |
| WinBox designation toggle ‚Äî clearing | Click on designated key icon ‚Üí calls `DELETE /api/v1/devices/{id}/winbox-profile` ‚Üí refreshes assignment list |
| Remove row ‚Äî confirmation | Inline confirmation in the same row card: `border-status-down/30 bg-status-down/10` container with "Keep Profile" + "Delete" buttons ‚Äî matches `SSHProfileManager` delete pattern |

### WinBox Action ‚Äî 3-State Logic

| State | Condition | Button | Tooltip (`title` attribute) |
|-------|-----------|--------|-----------------------------|
| 1 ‚Äî No profile | `!hasWinboxProfile` | `disabled` | "No WinBox profile designated" |
| 2 ‚Äî Profile set, bridge not running | `hasWinboxProfile && !bridgeRunning` | `disabled` | "WinBox bridge not running ‚Äî download from Settings" |
| 3 ‚Äî Enabled | `hasWinboxProfile && bridgeRunning` | enabled | none (no tooltip on enabled action) |

Source: CONTEXT.md D-10, D-11. Tooltip text is locked (exact strings from D-10).

"Has WinBox profile" is determined by: fetching `GET /api/v1/devices/{id}/credential-profiles` lazily when context menu opens (canvas) or on DeviceRow mount (table, background non-blocking). `is_winbox: true` on any assignment ‚Üí `hasWinboxProfile = true`.

## Implementation Notes for Executor

1. **No new packages.** Native `fetch` API handles bridge health check + bridge launch. All other HTTP calls use existing `requestJSON` / `requestJSONWithBody` helpers in `client.ts`.

2. **`ContextMenu` interface gap.** Add `title?: string` to the `ContextMenuItem` interface (`ContextMenu.tsx` line 4‚Äì11) and `title={item.title}` to the rendered `<button>`. Required for State 2 tooltip on the canvas WinBox item.

3. **Backend field mapping.** `assignedProfileResponse` serializes the profile ID as `"id"` (not `"profile_id"`). The `parseDeviceCredentialProfilesResponse` parser must map `item.id ‚Üí profile_id`.

4. **`useBridgeHealth` placement.** Instantiate locally in both `Canvas.tsx` and `Dashboard.tsx` (not in a Context provider). Both components are never mounted simultaneously (tab-based navigation), so no shared state is needed. Hook file: `frontend/src/hooks/useBridgeHealth.ts`.

5. **Virtual devices.** The WinBox context menu item is excluded for virtual devices ‚Äî same filter already applied to other items in `Canvas.tsx` `allItems` array.

6. **Context menu item order.** WinBox item placed after the Grafana item, before the interface-stats item (source: CONTEXT.md Claude's Discretion ‚Äî natural placement confirmed).

7. **`useBridgeHealth` ‚Äî silent failure.** Health check `catch` block calls `setBridgeRunning(false)` only ‚Äî no `console.error` or toast. Network errors on `localhost:1337` are expected when bridge is not running.

8. **"Has WinBox profile" state strategy.** Canvas: fetch device credential profiles lazily when context menu opens (per-device, per-open). DeviceRow: fetch on mount, background non-blocking, no loading indicator in the row. Store result in local state `hasWinboxProfile: boolean`.

9. **Cancel button labels.** Do not use the generic label "Cancel" anywhere. Delete profile confirmation cancel: "Keep Profile". Assignment add dismiss: "Dismiss". Form back navigation: back-arrow icon only (no text label).
